## What

`VillainTrail.tsx` renders the villain's "last seen" indicator on the map — a dimmed sprite showing where the villain was most recently spotted. It uses the `trailSprite` from the content pack's `villain` config (not the main villain sprite, which is a full-color character).

The component is almost identical to `PlayerMarker` but with reduced opacity to suggest the villain has already been there and moved on. Do not merge these into a single component — they have different semantic meaning and may diverge in future (e.g. VillainTrail may add a pulsing animation, trail sparkle, etc.).

## Tasks

1. Create `src/components/Map/VillainTrail.tsx`:
   ```tsx
   interface Position {
     x: number;
     y: number;
   }

   interface VillainTrailProps {
     position: Position;
     sprite: string;
   }

   export function VillainTrail({ position, sprite }: VillainTrailProps): React.ReactElement {
     return (
       <img
         src={sprite}
         alt=""
         aria-hidden="true"
         className="absolute w-8 h-8 translate-x-[-50%] translate-y-[-50%] pointer-events-none image-render-pixel opacity-50"
         style={{ left: position.x, top: position.y }}
       />
     );
   }
   ```

2. `opacity-50` gives the "ghost" effect — the villain was here but is gone. The content pack's `villain.trailSprite` (e.g. `assets/villain-trail.png`) may already be a faded variant; in that case `opacity-50` further dims it.

3. Like `PlayerMarker`, `position.x` / `position.y` are pixel values. The content pack stores the villain's last-seen node ID, and `MapCanvas` resolves that to a pixel coordinate before passing it here.

4. If no villain trail position is configured (e.g. the villain hasn't appeared yet), `MapScreen` should simply not render this component. Handle the conditional there, not here.
