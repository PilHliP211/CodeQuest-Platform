# S-04.05: MapCanvas Component

**Epic:** E-04
**Size:** S
**Dependencies:** S-04.01, S-04.02, S-04.03, S-04.04, S-03.05 (useContent hook from E-03)

## Skills to Load

- `react-component-structure` — file layout, named exports
- `content-pack-system` — reading map data from `useContent`
- `typescript-standards` — no `any`, typed content types
- `react-context-pattern` — consuming `useContent`

## What

`MapCanvas.tsx` is the main map container. It:

1. Reads map configuration from the content pack via `useContent()`
2. Renders the background image at the content-defined pixel dimensions
3. Renders an SVG overlay for all `MapEdge` connections
4. Renders all `MapNode` components at their content-defined positions
5. Derives node state (locked / available / completed) from the progress store — for now, stub all nodes as `'available'`

Coordinate system: node positions in the content pack are 0-1 normalized (`x: 0.82, y: 0.28`). To convert to pixel positions, multiply by `map.width` and `map.height` respectively. This component performs that conversion.

## Tasks

1. Create `src/components/Map/MapCanvas.tsx`:
   ```tsx
   import { useContent } from '@/engine/ContentContext';
   import { MapEdge } from './MapEdge';
   import { MapNode } from './MapNode';

   interface MapCanvasProps {
     onNodeSelect: (nodeId: string) => void;
   }

   export function MapCanvas({ onNodeSelect }: MapCanvasProps): React.ReactElement {
     const { course } = useContent();
     const { map } = course;

     // Stub: derive node state from progress store (E-12).
     // Until E-12 exists, all nodes are available.
     function getNodeState(_nodeId: string): 'locked' | 'available' | 'completed' {
       return 'available';
     }

     return (
       <div
         className="relative overflow-hidden"
         style={{ width: map.width, height: map.height }}
       >
         {/* Background image */}
         <img
           src={map.backgroundImage}
           alt="World map"
           className="absolute inset-0 w-full h-full object-cover image-render-pixel"
           draggable={false}
         />

         {/* SVG overlay for edges — pointer-events: none so clicks reach nodes */}
         <svg
           className="absolute top-0 left-0 pointer-events-none"
           width={map.width}
           height={map.height}
           aria-hidden="true"
         >
           {map.edges.map((edge) => {
             const fromNode = map.nodes.find((n) => n.id === edge.from);
             const toNode = map.nodes.find((n) => n.id === edge.to);
             if (fromNode === undefined || toNode === undefined) return null;
             return (
               <MapEdge
                 key={`${edge.from}-${edge.to}`}
                 from={{ x: fromNode.x * map.width, y: fromNode.y * map.height }}
                 to={{ x: toNode.x * map.width, y: toNode.y * map.height }}
               />
             );
           })}
         </svg>

         {/* Map nodes */}
         {map.nodes.map((node) => (
           <MapNode
             key={node.id}
             node={node}
             state={getNodeState(node.id)}
             onClick={() => onNodeSelect(node.id)}
           />
         ))}
       </div>
     );
   }
   ```

2. The `getNodeState` stub uses `_nodeId` (underscore prefix) to satisfy the linter's no-unused-vars rule while clearly signalling it is a placeholder. When E-12 ships, replace this with a real call to the progress store hook.

3. `map.backgroundImage` is a path relative to the content pack root. E-03's content loader is responsible for resolving asset paths correctly for the Vite build. Do not prefix the path here.

4. The node's `onClick` always calls `onNodeSelect` — `MapNode` itself handles whether to fire it based on its `state` prop (S-04.01 uses `disabled` for non-available nodes).

5. `draggable={false}` on the background image prevents the browser's default drag-image behavior from activating when the player drags across the map.

## Done When

- [ ] Background image renders at content-defined pixel dimensions
- [ ] All nodes from `map.nodes` rendered at correct percentage positions
- [ ] All edges from `map.edges` rendered as SVG lines connecting the right nodes
- [ ] Edges whose `from` or `to` node ID is not found are silently skipped (defensive)
- [ ] `onNodeSelect` is wired to each node's click handler
- [ ] Node state stub returns `'available'` for all nodes
- [ ] SVG overlay has `pointer-events: none` so node buttons receive click events
- [ ] No `any` types, no `!` non-null assertions
