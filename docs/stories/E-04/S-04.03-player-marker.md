## What

`PlayerMarker.tsx` renders the player's sprite at a given pixel position on the map canvas. It shows where the player currently is. The position is already converted to pixel coordinates by the caller (`MapCanvas`) — this component just places the image.

The marker is `position: absolute` inside the `position: relative` map container. The sprite is centered on the coordinate point using negative translate.

## Tasks

1. Create `src/components/Map/PlayerMarker.tsx`:
   ```tsx
   interface Position {
     x: number;
     y: number;
   }

   interface PlayerMarkerProps {
     position: Position;
     sprite: string;
   }

   export function PlayerMarker({ position, sprite }: PlayerMarkerProps): React.ReactElement {
     return (
       <img
         src={sprite}
         alt=""
         aria-hidden="true"
         className="absolute w-8 h-8 translate-x-[-50%] translate-y-[-50%] pointer-events-none image-render-pixel"
         style={{ left: position.x, top: position.y }}
       />
     );
   }
   ```

2. The `alt=""` and `aria-hidden="true"` are intentional — the marker is decorative. The map node labels and their ARIA labels (S-04.01) communicate the player's location semantically.

3. `pointer-events-none` prevents the sprite from intercepting clicks that should reach the `MapNode` buttons beneath it.

4. `image-render-pixel` is a custom Tailwind utility added in E-01 (`image-rendering: pixelated`) — do not use `style={{ imageRendering: 'pixelated' }}` inline if the utility already exists.

5. `position.x` and `position.y` are **pixel values**, not percentages. `MapCanvas` converts 0-1 normalized coordinates from the content pack to pixel values before passing them here.
