## What

The lesson runner is a `useReducer`-based state machine. Each action advances the state through a well-defined set of transitions. There are no intermediate "ambiguous" states — at any point, the component tree can read `lessonState` and know exactly what to render.

Phase transition flow:
- `START_LESSON` → Phase 1 `narrative-intro` (shows intro narrative)
- `NARRATIVE_COMPLETE` → `challenge` (player starts coding)
- `RUN_CODE` → `running` (disable run button)
- `RUN_SUCCESS` (Phase 1/2) → `success` then `narrative-outro` then `NEXT_PHASE`
- `RUN_SUCCESS` (Phase 3) → `success` then `COMPLETE_LESSON`
- `RUN_FAIL` → `failed` (show error, increment attempt count)
- `NEXT_PHASE` → next phase's `narrative-intro`
- `COMPLETE_LESSON` → `'complete'` phase

## Tasks

1. Create `src/engine/lessonRunner.ts`:
   ```ts
   import type { LessonState, LessonAction, PhaseStatus } from '@/types/lessonState';

   export function createInitialState(lessonId: string): LessonState {
     return {
       lessonId,
       currentPhase: 1,
       phaseStatus: 'narrative-intro',
       attemptCount: 0,
       hintsUsed: 0,
       syntaxUnlocked: false,
       lastError: null,
       pendingNarrative: null,
     };
   }

   export function lessonReducer(state: LessonState, action: LessonAction): LessonState {
     switch (action.type) {
       case 'START_LESSON':
         return createInitialState(action.lessonId);

       case 'NARRATIVE_COMPLETE':
         if (state.phaseStatus === 'narrative-intro') {
           return { ...state, phaseStatus: 'challenge', pendingNarrative: null };
         }
         if (state.phaseStatus === 'narrative-outro') {
           if (state.currentPhase === 'complete') return state;
           return { ...state, phaseStatus: 'challenge', pendingNarrative: null };
         }
         return state;

       case 'RUN_CODE':
         if (state.phaseStatus !== 'challenge' && state.phaseStatus !== 'failed') return state;
         return { ...state, phaseStatus: 'running', lastError: null };

       case 'RUN_SUCCESS': {
         if (state.currentPhase === 3) {
           return { ...state, phaseStatus: 'success' };
         }
         const nextPhase = (state.currentPhase + 1) as 1 | 2 | 3;
         return {
           ...state,
           phaseStatus: 'success',
           syntaxUnlocked: state.currentPhase === 2 ? true : state.syntaxUnlocked,
         };
       }

       case 'RUN_FAIL':
         return {
           ...state,
           phaseStatus: 'failed',
           attemptCount: state.attemptCount + 1,
           lastError: action.error,
         };

       case 'SHOW_HINT':
         return { ...state, hintsUsed: state.hintsUsed + 1 };

       case 'NEXT_PHASE': {
         if (state.currentPhase === 3 || state.currentPhase === 'complete') return state;
         const next = (state.currentPhase + 1) as 2 | 3;
         return {
           ...state,
           currentPhase: next,
           phaseStatus: 'narrative-intro',
           attemptCount: 0,
           lastError: null,
           pendingNarrative: null,
         };
       }

       case 'COMPLETE_LESSON':
         return { ...state, currentPhase: 'complete', phaseStatus: 'success' };

       case 'EXIT_LESSON':
         return createInitialState('');

       default:
         return state;
     }
   }
   ```

2. The narrative content to show during `narrative-intro` and `narrative-outro` is set via `pendingNarrative` — this is populated in S-11.04. For now, `pendingNarrative` starts as `null` and transitions to `challenge` immediately when `NARRATIVE_COMPLETE` is dispatched.

3. The `syntaxUnlocked` flag is set `true` when Phase 2 completes. S-11.05 (XP) and E-12 (persistence) read this to save the unlock state.
