# S-11.05: XP Award on Phase/Lesson Completion

**Epic:** E-11
**Size:** XS
**Dependencies:** S-11.02 (reducer), S-12.02 (progressStore, for writing XP)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `unit-tests` — Vitest AAA+O pattern for the pure logic added here
- `content-pack-system` — `course.xp` field: `{ phaseComplete, lessonComplete, hintUsed, syntaxEditorUsed }`
- `localstorage-pattern` — `progressStore.addXP()` from E-12

## What

XP is awarded at two moments: when each phase completes (`phaseComplete` XP), and when the full lesson completes (`lessonComplete` XP). Hints deduct XP (`hintUsed` × `hintsUsed`). Using the syntax editor for the first time in a lesson awards a bonus (`syntaxEditorUsed`).

This is a calculation + write story — it doesn't add new state, it reads existing state and calls `progressStore.addXP`.

## Tasks

1. Create `src/engine/xpCalculator.ts`:
   ```ts
   import type { XpConfig } from '@/types/content';
   import type { LessonState } from '@/types/lessonState';

   export function calculatePhaseXP(xpConfig: XpConfig, hintsUsed: number): number {
     return Math.max(0, xpConfig.phaseComplete + xpConfig.hintUsed * hintsUsed);
   }

   export function calculateLessonXP(
     xpConfig: XpConfig,
     hintsUsed: number,
     syntaxUnlocked: boolean,
   ): number {
     const base = xpConfig.lessonComplete;
     const hintPenalty = xpConfig.hintUsed * hintsUsed;
     const syntaxBonus = syntaxUnlocked ? (xpConfig.syntaxEditorUsed ?? 0) : 0;
     return Math.max(0, base + hintPenalty + syntaxBonus);
   }
   ```
   Note: `hintUsed` XP in `course.xp` is typically negative (e.g. `-5`), so `hintPenalty` naturally reduces the total.

2. In `LessonContext` (S-11.06), after dispatching `RUN_SUCCESS` for Phase 3, calculate and write XP:
   ```ts
   const phaseXP = calculatePhaseXP(course.xp, lessonState.hintsUsed);
   progressStore.addXP(phaseXP);
   ```
   And on `COMPLETE_LESSON`:
   ```ts
   const lessonXP = calculateLessonXP(course.xp, lessonState.hintsUsed, lessonState.syntaxUnlocked);
   progressStore.addXP(lessonXP);
   ```

3. XP is never negative. `Math.max(0, ...)` ensures a bad run with many hints still awards 0, not a negative number.

4. `XpConfig` type: `{ phaseComplete: number; lessonComplete: number; hintUsed: number; syntaxEditorUsed: number }` — already defined in `src/types/content.ts` (E-03).
