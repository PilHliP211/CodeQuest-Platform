# S-11.01: Lesson State Types

**Epic:** E-11
**Size:** XS
**Dependencies:** S-03.01 (content types)

## Skills to Load

- `typescript-standards` — discriminated unions, no `any`
- `naming-conventions` — type file in `src/types/`

## What

Define the TypeScript types for the lesson runner's state. These types are the contract between the state machine (S-11.02) and every component that consumes it.

## Tasks

1. Create `src/types/lessonState.ts`:
   ```ts
   export type LessonPhase = 1 | 2 | 3 | 'complete';

   export type PhaseStatus =
     | 'narrative-intro'   // Showing the phase intro narrative
     | 'challenge'         // Player is solving the challenge
     | 'running'           // Code is executing
     | 'success'           // Challenge solved, showing success state
     | 'narrative-outro'   // Showing the phase outro/reveal narrative
     | 'failed';           // Last run failed, showing error

   export interface LessonState {
     lessonId: string;
     currentPhase: LessonPhase;
     phaseStatus: PhaseStatus;
     attemptCount: number;        // Resets each time a new phase starts
     hintsUsed: number;           // Cumulative for the whole lesson
     syntaxUnlocked: boolean;     // Set true when Phase 2 completes
     lastError: string | null;    // Player-friendly error from last failed run
     pendingNarrative: NarrativeScene[] | null; // Set when a narrative should play
   }
   ```

2. Import `NarrativeScene` from `src/types/content.ts` (defined in E-03/S-03.01).

3. Define the action union:
   ```ts
   export type LessonAction =
     | { type: 'START_LESSON'; lessonId: string }
     | { type: 'NARRATIVE_COMPLETE' }
     | { type: 'RUN_CODE' }
     | { type: 'RUN_SUCCESS' }
     | { type: 'RUN_FAIL'; error: string }
     | { type: 'SHOW_HINT' }
     | { type: 'NEXT_PHASE' }
     | { type: 'COMPLETE_LESSON' }
     | { type: 'EXIT_LESSON' };
   ```
