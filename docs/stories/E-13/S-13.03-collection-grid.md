## What

`CollectionGrid` renders all flag tiles in a responsive grid. It gets the full node list from the content pack (so it knows what flags *could* be collected) and the collected flag list from `deriveCollectedFlags`. It shows a counter: "X / Y Flags Collected".

## Tasks

1. Create `src/components/Collection/CollectionGrid.tsx`:
   ```tsx
   import { useContent } from '@/engine/ContentContext';
   import { deriveCollectedFlags } from '@/engine/collectionDeriver';
   import { FlagTile } from './FlagTile';

   export function CollectionGrid(): React.ReactElement {
     const { course } = useContent();
     const collectedFlags = deriveCollectedFlags(course.map.nodes, course.id);
     const collectedById = Object.fromEntries(collectedFlags.map((f) => [f.lessonId, f]));

     const total = course.map.nodes.length;
     const collected = collectedFlags.length;

     return (
       <section aria-label="Flag Collection">
         <h2 className="font-pixel text-lg text-white mb-4">Flag Collection</h2>
         <p className="font-pixel text-sm text-yellow-300 mb-6">
           {collected} / {total} Flags Collected
         </p>
         <div className="grid grid-cols-3 gap-4 sm:grid-cols-4 md:grid-cols-6">
           {course.map.nodes.map((node) => (
             <FlagTile
               key={node.id}
               node={node}
               collected={collectedById[node.id] ?? null}
             />
           ))}
         </div>
       </section>
     );
   }
   ```

2. The grid is responsive: 3 columns on mobile, 4 on sm, 6 on md+. This adapts to how many lessons are in the pack.

3. The node order from `course.map.nodes` is used as-is — content pack authors control the display order by ordering nodes in the JSON.

4. `deriveCollectedFlags` is a synchronous call — no loading state needed.
