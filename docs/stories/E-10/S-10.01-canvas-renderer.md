# S-10.01: CanvasRenderer Component

**Epic:** E-10
**Size:** XS
**Dependencies:** S-01.05 (folder structure), S-09.01 (interpreter types — understanding the execution pipeline)

## Skills to Load

- `react-component-structure` — file layout, named exports, ref forwarding
- `typescript-standards` — typed imperative handle with `useImperativeHandle`
- `naming-conventions` — component lives in `src/renderer/`

## What

Create the `CanvasRenderer` React component. It renders a `<canvas>` element sized to the lesson's `phase3.canvas.width` and `phase3.canvas.height`. It exposes an imperative handle so parent components can call `clear()` and `getContext()` without drilling through props.

Device pixel ratio scaling ensures crisp rendering on high-DPI screens: the canvas backing buffer is scaled up, but CSS dimensions stay at the logical size. Combined with `image-rendering: pixelated`, this produces a sharp pixel-art aesthetic.

## Tasks

1. Create `src/renderer/CanvasRenderer.tsx`:
   ```tsx
   import { forwardRef, useEffect, useImperativeHandle, useRef } from 'react';

   export interface CanvasRendererHandle {
     clear(): void;
     getContext(): CanvasRenderingContext2D;
   }

   interface CanvasRendererProps {
     width: number;
     height: number;
     className?: string;
   }

   export const CanvasRenderer = forwardRef<CanvasRendererHandle, CanvasRendererProps>(
     function CanvasRenderer({ width, height, className }, ref) {
       const canvasRef = useRef<HTMLCanvasElement>(null);

       useEffect(() => {
         const canvas = canvasRef.current;
         if (canvas === null) return;
         const dpr = window.devicePixelRatio ?? 1;
         canvas.width = width * dpr;
         canvas.height = height * dpr;
         const ctx = canvas.getContext('2d');
         if (ctx === null) return;
         ctx.scale(dpr, dpr);
       }, [width, height]);

       useImperativeHandle(ref, () => ({
         clear() {
           const canvas = canvasRef.current;
           if (canvas === null) return;
           const ctx = canvas.getContext('2d');
           if (ctx === null) return;
           ctx.clearRect(0, 0, width, height);
         },
         getContext() {
           const canvas = canvasRef.current;
           if (canvas === null) throw new Error('Canvas not mounted');
           const ctx = canvas.getContext('2d');
           if (ctx === null) throw new Error('Could not get 2D context');
           return ctx;
         },
       }));

       return (
         <canvas
           ref={canvasRef}
           style={{
             width,
             height,
             imageRendering: 'pixelated',
           }}
           className={className}
         />
       );
     },
   );
   ```

2. The `useEffect` runs when `width` or `height` changes, re-applying DPR scaling if the lesson canvas dimensions differ between lessons.

3. CSS `width`/`height` are set via the `style` prop (not Tailwind) so they exactly match the logical dimensions regardless of Tailwind's sizing scale.
