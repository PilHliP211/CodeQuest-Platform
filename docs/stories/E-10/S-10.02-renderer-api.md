# S-10.02: rendererAPI — Drawing Functions

**Epic:** E-10
**Size:** M
**Dependencies:** S-10.01 (CanvasRenderer, needs `CanvasRenderingContext2D`)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `unit-tests` — Vitest AAA+O pattern for the pure logic added here
- `typescript-standards` — no `any`, typed return objects, `unknown` + type guards for validation
- `naming-conventions` — `rendererAPI.ts` lives in `src/renderer/`
- `security-rules` — color validation via type guard, not arbitrary string passthrough
- `error-handling` — drawing functions fail silently (return early) on invalid input

## What

Create `src/renderer/rendererAPI.ts`. This module is the bridge between the interpreter and the HTML5 canvas. It exports `createRendererAPI(ctx)` which returns a `RendererAPI` object containing all drawing functions.

Every call made through `RendererAPI` is recorded in a `callLog` array — this is the data that `successEvaluator.ts` (S-10.05) compares against the lesson solution. The call log captures the function name and arguments exactly as called.

All five drawing functions are implemented here, even if only a subset are exposed to any given lesson. The whitelist enforcement (which functions actually get injected into the interpreter) is a separate concern handled in S-10.03.

## Tasks

1. Define the `RendererCall` and `RendererAPI` types:
   ```ts
   export interface RendererCall {
     fn: string;
     args: (string | number)[];
   }

   export interface RendererAPI {
     fillBackground(color: string): void;
     drawRect(x: number, y: number, width: number, height: number, color: string): void;
     drawCircle(x: number, y: number, radius: number, color: string): void;
     drawStripe(x: number, y: number, width: number, height: number, color: string, angle?: number): void;
     drawTriangle(x1: number, y1: number, x2: number, y2: number, x3: number, y3: number, color: string): void;
     getCallLog(): RendererCall[];
     clearCallLog(): void;
   }
   ```

2. Implement `createRendererAPI(ctx: CanvasRenderingContext2D): RendererAPI`:
   - `fillBackground(color)` — calls `ctx.fillStyle = color; ctx.fillRect(0, 0, canvas.width, canvas.height)`. Record `{ fn: 'fillBackground', args: [color] }`.
   - `drawRect(x, y, width, height, color)` — `ctx.fillStyle = color; ctx.fillRect(x, y, width, height)`. Record args.
   - `drawCircle(x, y, radius, color)` — `ctx.beginPath(); ctx.arc(x, y, radius, 0, Math.PI * 2); ctx.fillStyle = color; ctx.fill()`. Record args.
   - `drawStripe(x, y, width, height, color, angle?)` — if `angle` is defined and non-zero, use canvas transform to rotate around the stripe center before drawing; restore transform after. Record all args including angle.
   - `drawTriangle(x1, y1, x2, y2, x3, y3, color)` — `ctx.beginPath(); ctx.moveTo(x1,y1); ctx.lineTo(x2,y2); ctx.lineTo(x3,y3); ctx.closePath(); ctx.fillStyle = color; ctx.fill()`. Record args.

3. Each drawing function validates its color argument using a type guard before calling `ctx.fillStyle`:
   ```ts
   function isValidColor(value: unknown): value is string {
     return typeof value === 'string' && value.trim().length > 0;
   }
   ```
   If validation fails, return early without drawing or logging.

4. `getCallLog()` returns a copy of the internal array (`[...callLog]`) so callers cannot mutate it.

5. `clearCallLog()` empties the internal log array.

6. The canvas dimensions needed by `fillBackground` should be read from `ctx.canvas.width` and `ctx.canvas.height` — do not rely on the CSS logical dimensions (the backing buffer is DPR-scaled).
