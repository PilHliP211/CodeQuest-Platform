# S-10.06: "Reset Canvas" Button

**Epic:** E-10
**Size:** XS
**Dependencies:** S-10.04 (sequential animation, cancellable)

## Skills to Load

- `react-component-structure` — simple button with side effects
- `accessibility` — button label, focus management

## What

A "Reset ↺" button below the canvas clears the drawing surface, resets the `callLog` in `rendererAPI`, and cancels any in-progress drawing animation. This lets the player retry their code from a clean slate without reloading the page.

## Tasks

1. Add a `reset()` method to the `RendererAPI` interface and implementation:
   ```ts
   reset(): void {
     ctx.clearRect(0, 0, ctx.canvas.width, ctx.canvas.height);
     callLog.length = 0; // mutate in place to clear the log
   }
   ```

2. In `CanvasRenderer.tsx`, expose a `resetRef` (or use `useImperativeHandle`) that the parent can call to cancel any in-progress animation and reset the canvas:
   ```tsx
   export interface CanvasRendererHandle {
     reset: () => void;
   }

   export const CanvasRenderer = forwardRef<CanvasRendererHandle, CanvasRendererProps>(
     function CanvasRenderer({ width, height, api }, ref) {
       useImperativeHandle(ref, () => ({
         reset() {
           api.reset();
           // Cancel any pending animation timeouts (implementation detail from S-10.04)
         },
       }));
       // ...
     }
   );
   ```

3. Render the reset button adjacent to the canvas:
   ```tsx
   <button
     type="button"
     onClick={() => canvasRef.current?.reset()}
     className="font-pixel text-sm mt-2"
     aria-label="Reset canvas"
   >
     Reset ↺
   </button>
   ```

4. After reset, the success evaluator starts fresh — the `callLog` is empty so any previous successful run is no longer counted. The player must re-run their code to succeed.

## Done When

- [ ] "Reset ↺" button renders below the canvas
- [ ] Clicking reset calls `ctx.clearRect` on the full canvas
- [ ] `callLog` is cleared on reset
