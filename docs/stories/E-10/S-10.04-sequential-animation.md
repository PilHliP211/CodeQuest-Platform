# S-10.04: Sequential Animation

**Epic:** E-10
**Size:** S
**Dependencies:** S-10.03 (whitelist scope), S-09.07 (step-by-step execution mode from interpreter)

## What

Instead of executing all drawing calls synchronously and updating the canvas in a single frame, the renderer plays them back one at a time with a 150 ms delay between each. This makes the flag "paint itself" step by step — a satisfying visual reveal that reinforces the connection between each line of code and its visual output.

The step-by-step execution mode from E-09 (S-09.07) yields one drawing call at a time. For each yielded step, the renderer awaits a 150 ms delay before executing the next. A cancellation token allows the "Reset ↺" button (S-10.06) to abort an in-progress animation cleanly.

## Tasks

1. Create `src/renderer/animateDrawing.ts`:
   ```ts
   export interface AnimationHandle {
     cancel(): void;
   }

   export function animateDrawing(
     steps: Generator<() => void>,
     delayMs = 150,
   ): AnimationHandle {
     let cancelled = false;

     async function run(): Promise<void> {
       for (const step of steps) {
         if (cancelled) break;
         step();
         await delay(delayMs);
       }
     }

     void run();

     return {
       cancel() {
         cancelled = true;
       },
     };
   }

   function delay(ms: number): Promise<void> {
     return new Promise((resolve) => setTimeout(resolve, ms));
   }
   ```

2. The `steps` generator comes from the interpreter's step-by-step execution mode. Each yielded value is a zero-argument function that, when called, executes one drawing command against the canvas context.

3. In the Phase 3 run handler (wired up in E-11):
   - Call `api.clearCallLog()` before starting animation
   - Obtain the step generator from the interpreter
   - Call `animateDrawing(steps)` and store the returned `AnimationHandle`
   - On "Reset ↺" click, call `handle.cancel()` before clearing the canvas

4. The `animateDrawing` function uses `void run()` to fire the async loop without returning a promise to the call site — the `AnimationHandle` is the only control surface. This avoids unhandled promise rejection warnings while keeping the API simple.

5. If the interpreter produces no steps (empty program), `animateDrawing` returns immediately — no animation, no error.

## Done When

- [ ] `src/renderer/animateDrawing.ts` exported and typed

- [ ] `cancel()` stops execution after the current in-progress step — does not leave the canvas in a half-drawn state beyond the step boundary

- [ ] Empty generator completes immediately without delay
