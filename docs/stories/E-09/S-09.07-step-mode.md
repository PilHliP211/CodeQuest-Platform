# S-09.07: Step-by-Step Execution Mode

**Epic:** E-09
**Size:** S
**Dependencies:** S-09.04 (`interpreter.ts`)

## Skills to Load

- `typescript-standards` ‚Äî generator functions, typed yield values
- `architecture-overview` ‚Äî step mode is consumed by E-10 canvas animation

## What

The canvas renderer (E-10) needs to animate each drawing call sequentially ‚Äî a brief pause between `fillBackground` and `drawCircle` makes the flag "build" before the player's eyes. This requires the interpreter to yield control after each top-level function call so the renderer can insert delays.

This story adds a generator-based step mode to the interpreter. In step mode, `executeStep` is a generator that yields after each `ExpressionStatement`. The canvas renderer iterates it with `for...of` (or async iteration with `await delay(ms)` between yields).

## Tasks

1. Add `ExecutionStep` type and `executeStep` generator to `interpreter.ts`:
   ```ts
   export interface ExecutionStep {
     type: 'call';
     fnName: string;
     line?: number;
   }

   export function* executeStep(
     code: string,
     scope: Record<string, unknown>,
   ): Generator<ExecutionStep, ExecutionResult, void> {
     let ast: acorn.Node;
     try {
       ast = acorn.parse(code, { ecmaVersion: 2020, locations: true });
     } catch (err) {
       return { success: false, error: parseFriendlyError(err) };
     }

     try {
       validateAST(ast as unknown as ASTNode, Object.keys(scope));
     } catch (err) {
       if (err instanceof SandboxError) {
         return { success: false, error: { message: err.message, line: err.line } };
       }
       throw err;
     }

     const env = { ...scope };
     const body = (ast as { body: ASTNode[] }).body;

     for (const stmt of body) {
       if (stmt.type === 'ExpressionStatement' && stmt.expression.type === 'CallExpression') {
         const callee = stmt.expression.callee;
         const fnName = callee.type === 'Identifier' ? callee.name : '(unknown)';
         yield { type: 'call', fnName, line: stmt.loc?.start.line };
       }
       try {
         executeNode(stmt, env);
       } catch (err) {
         if (err instanceof RuntimeError) {
           return { success: false, error: { message: err.message, line: err.line } };
         }
         throw err;
       }
     }

     return { success: true };
   }
   ```

2. The canvas renderer (E-10, S-10.04) uses this generator like:
   ```ts
   const gen = executeStep(code, scope);
   for (const step of gen) {
     await delay(150); // pause between drawing calls
   }
   const result = gen.return(undefined).value; // get final ExecutionResult
   ```
   This story just implements the generator; E-10 wires it up.

3. The existing `execute()` function is unchanged ‚Äî it still runs synchronously. Step mode is an optional alternative, not a replacement.

4. Export `executeStep` and `ExecutionStep` from `src/engine/interpreter/index.ts`.
