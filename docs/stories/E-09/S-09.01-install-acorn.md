**Epic:** E-09
**Size:** XS
**Dependencies:** S-01.05 (folder structure), `dependency-policy` skill

## Skills to Load

- `testing-strategy` — background on the outcome-focused validation philosophy; this story adds no executable code, so no new tests are required
- `dependency-policy` — evaluate the package before adding it
- `typescript-standards` — verify `@types/acorn` is available (it's built into acorn v8+)

## What

`acorn` is a battle-tested, standards-compliant JavaScript parser that produces ESTree-compatible ASTs. It's the same parser used by Babel, Webpack, and Rollup internally — well-maintained and tiny (~200KB). This story installs it and verifies the TypeScript types are available.

## Tasks

1. Install:
   ```
   npm install acorn
   ```
   Note: acorn v8+ ships its own TypeScript types — no separate `@types/acorn` needed.

2. Verify the import works:
   ```ts
   import * as acorn from 'acorn';
   const ast = acorn.parse('let x = 1;', { ecmaVersion: 2020 });
   ```

3. Create the interpreter directory: `src/engine/interpreter/`.

4. Create a minimal `src/engine/interpreter/index.ts` that re-exports from `interpreter.ts` (to be created in S-09.04). For now, just the empty module:
   ```ts
   // Entry point for the interpreter module — filled in by S-09.04
   export {};
   ```

5. Document in a comment why we chose `acorn` over alternatives:
   - `eval()` / `Function()` — forbidden by platform hard rules
   - `vm2` — Node.js only, not browser-compatible
   - QuickJS WASM — preferred stretch goal (E-18), too complex for v1
   - `acorn` — browser-safe parser, zero runtime dependencies
