# S-09.06: Interpreter Unit Tests

**Epic:** E-09
**Size:** M
**Dependencies:** S-09.05 (full interpreter + errors)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `unit-tests` — example-based unit tests for the specific cases called out in this story
- `invariant-tests` — fast-check property tests for the security boundary
- `typescript-standards` — typed test helpers
- `ci-and-quality-gates` — tests must pass in `npm run lint` + CI

## What

The interpreter is the security-critical path. It needs a comprehensive test suite covering the happy path, the sandbox boundary, and error cases. Tests also serve as a living specification for what the simplified-JS subset supports.

Tests use Vitest (included with the Vite setup from E-01).

## Tasks

1. Create `src/engine/interpreter/interpreter.test.ts`:

   ```ts
   import { describe, it, expect, vi } from 'vitest';
   import { execute } from './interpreter';

   describe('interpreter — happy path', () => {
     it('calls a single function', () => {
       const fn = vi.fn();
       execute('doThing()', { doThing: fn });
       expect(fn).toHaveBeenCalledOnce();
     });

     it('calls multiple functions in order', () => {
       const log: string[] = [];
       const scope = {
         a: () => log.push('a'),
         b: () => log.push('b'),
         c: () => log.push('c'),
       };
       execute('a()\nb()\nc()', scope);
       expect(log).toEqual(['a', 'b', 'c']);
     });

     it('passes string arguments', () => {
       const fn = vi.fn();
       execute('fill("red")', { fill: fn });
       expect(fn).toHaveBeenCalledWith('red');
     });

     it('passes numeric arguments', () => {
       const fn = vi.fn();
       execute('draw(10, 20, 5)', { draw: fn });
       expect(fn).toHaveBeenCalledWith(10, 20, 5);
     });

     it('let declaration and use', () => {
       const fn = vi.fn();
       execute('let x = 42\nfn(x)', { fn });
       expect(fn).toHaveBeenCalledWith(42);
     });

     it('for loop runs N times', () => {
       const fn = vi.fn();
       execute('for (let i = 0; i < 3; i++) { fn() }', { fn });
       expect(fn).toHaveBeenCalledTimes(3);
     });

     it('if/else — true branch', () => {
       const yes = vi.fn();
       const no = vi.fn();
       execute('if (1 > 0) { yes() } else { no() }', { yes, no });
       expect(yes).toHaveBeenCalledOnce();
       expect(no).not.toHaveBeenCalled();
     });

     it('function declaration and call', () => {
       const fn = vi.fn();
       execute('function go() { fn() }\ngo()\ngo()', { fn });
       expect(fn).toHaveBeenCalledTimes(2);
     });
   });

   describe('interpreter — sandbox security', () => {
     it('rejects window access', () => {
       const result = execute('window.alert("x")', {});
       expect(result.success).toBe(false);
     });

     it('rejects document access', () => {
       const result = execute('document.write("x")', {});
       expect(result.success).toBe(false);
     });

     it('rejects eval', () => {
       const result = execute('eval("1+1")', {});
       expect(result.success).toBe(false);
     });

     it('rejects localStorage', () => {
       const result = execute('localStorage.getItem("x")', {});
       expect(result.success).toBe(false);
     });

     it('rejects var declaration', () => {
       const result = execute('var x = 1', {});
       expect(result.success).toBe(false);
     });

     it('rejects import', () => {
       const result = execute('import x from "y"', {});
       expect(result.success).toBe(false);
     });
   });

   describe('interpreter — errors', () => {
     it('syntax error returns friendly message with line number', () => {
       const result = execute('let x = (', {});
       expect(result.success).toBe(false);
       if (!result.success) {
         expect(result.error.message).toContain('Syntax error');
       }
     });

     it('unknown function returns friendly message', () => {
       const result = execute('unknownFn()', {});
       expect(result.success).toBe(false);
       if (!result.success) {
         expect(result.error.message).toContain('unknownFn');
       }
     });

     it('loop exceeding limit returns error', () => {
       const result = execute('for (let i = 0; i < 100000; i++) {}', {});
       expect(result.success).toBe(false);
       if (!result.success) {
         expect(result.error.message).toContain('too many times');
       }
     });

     it('empty code returns success', () => {
       const result = execute('', {});
       expect(result.success).toBe(true);
     });
   });
   ```

2. Run tests: `npx vitest run src/engine/interpreter/interpreter.test.ts`. All tests must pass.

3. The sandbox security tests depend on the sandbox actually rejecting the code — not on the runtime throwing. Verify each rejects at the `validateAST` stage (before execution).
