# S-15.03: Smoke Test at Live URL

**Epic:** E-15
**Size:** XS
**Dependencies:** S-15.02 (deploy workflow pushed and run)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `e2e-tests` — Playwright end-to-end for this critical learner journey
- `ci-and-quality-gates` — production smoke test checklist

## What

After the first successful deploy, verify the app is live and functional at `byram.dev/codequest-platform/`. This is a one-time manual verification checklist.

## One-Time GitHub Setup (before first deploy)

1. In GitHub repository Settings → Pages:
   - Source: "Deploy from a branch"
   - Branch: `gh-pages` / `/ (root)`
   - Save

2. The `gh-pages` branch is created automatically on the first deploy run. It does not need to be created manually.

3. If using a custom domain (`byram.dev`): the DNS is already configured on the personal-website repo. Project pages at subpaths (`/codequest-platform/`) work automatically — no additional DNS config needed.

## Smoke Test Checklist

After the deploy workflow completes (check Actions tab in GitHub):

- [ ] Visit `https://byram.dev/codequest-platform/` — page loads (no 404 or blank screen)
- [ ] Browser DevTools Network tab shows no 404 errors for assets
- [ ] Name entry screen appears (first visit)
- [ ] Enter a name — main app renders
- [ ] Map screen visible with Japan node
- [ ] Page title shows "CodeQuest" (or whatever is set in `index.html`)
- [ ] No console errors in DevTools
- [ ] App works on mobile viewport (responsive)

## If the Deploy Fails

Common issues:
- **Build fails in CI**: Check the Actions log for TypeScript or lint errors
- **Assets 404 after deploy**: Verify `base: '/codequest-platform/'` in `vite.config.ts`
- **Blank page**: Open DevTools → Console — likely a JS error. Check asset paths
- **`gh-pages` branch not created**: First deploy creates it. Check that the workflow ran successfully
- **Pages not enabled**: Go to repo Settings → Pages and enable it

## Done When

- [ ] All smoke test checkboxes above are checked
- [ ] App is live at `byram.dev/codequest-platform/`
- [ ] No errors in browser console on first load
