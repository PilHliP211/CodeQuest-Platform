# S-15.01: Verify Vite Base Path Configuration

**Epic:** E-15
**Size:** XS
**Dependencies:** S-01.15 (Vite base path — already configured)

## Skills to Load

- `ci-and-quality-gates` — build verification
- `architecture-overview` — deployment target is `byram.dev/codequest-platform/`

## What

S-01.15 set `base: '/codequest-platform/'` in `vite.config.ts`. This story verifies that the production build correctly uses this base path for all asset URLs, and that the app functions when served from a subpath (not the root).

## Tasks

1. Run `npm run build` and inspect `dist/index.html`:
   ```
   grep 'src=' dist/index.html
   grep 'href=' dist/index.html
   ```
   All asset paths should start with `/codequest-platform/assets/`, not `/assets/`.

2. Serve the build locally to verify subpath serving works:
   ```
   npx serve dist --single
   ```
   Then visit `http://localhost:3000/codequest-platform/` (adjust port). The app should load without 404 errors on assets.

   Alternatively, use `vite preview` which respects the base path automatically.

3. Verify `vite.config.ts` has:
   ```ts
   export default defineConfig({
     base: '/codequest-platform/',
     // ... rest of config
   });
   ```
   If not already set (should be from S-01.15), add it now.

4. Check that React Router (if used) or any `<a href>` links use relative paths or the Vite `base` properly. For v1 with no router, this is a non-issue.

## Done When

- [ ] `npm run build` succeeds
- [ ] `dist/index.html` references `/codequest-platform/assets/` not `/assets/`
- [ ] App loads without 404 errors when served from the `/codequest-platform/` subpath
- [ ] `vite.config.ts` has `base: '/codequest-platform/'`
