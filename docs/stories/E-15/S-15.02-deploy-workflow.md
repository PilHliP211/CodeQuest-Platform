# S-15.02: GitHub Actions Deploy Workflow

**Epic:** E-15
**Size:** S
**Dependencies:** S-15.01 (build verified), S-01.18 (CI workflow — different file, do not modify)

## Skills to Load

- `testing-strategy` — background on the outcome-focused validation philosophy; this story adds no executable code, so no new tests are required
- `ci-and-quality-gates` — GitHub Actions workflow syntax
- `dependency-policy` — use pinned action versions

## What

Create `.github/workflows/deploy.yml`. This workflow triggers on push to `main`, builds the app, and deploys the `dist/` directory to the `gh-pages` branch using the `peaceiris/actions-gh-pages` action. The `gh-pages` branch is what GitHub Pages serves.

## Tasks

1. Create `.github/workflows/deploy.yml`:
   ```yaml
   name: Deploy to GitHub Pages

   on:
     push:
       branches:
         - main

   permissions:
     contents: write

   jobs:
     deploy:
       runs-on: ubuntu-latest

       steps:
         - name: Checkout
           uses: actions/checkout@v4

         - name: Set up Node
           uses: actions/setup-node@v4
           with:
             node-version-file: '.nvmrc'  # or specify: node-version: '20'
             cache: 'npm'

         - name: Install dependencies
           run: npm ci

         - name: Build
           run: npm run build

         - name: Deploy to gh-pages
           uses: peaceiris/actions-gh-pages@v3
           with:
             github_token: ${{ secrets.GITHUB_TOKEN }}
             publish_dir: ./dist
   ```

2. Note on `node-version-file`: if `.nvmrc` doesn't exist, use `node-version: '20'` (or whatever version is in `package.json` `engines` field). Check the existing CI workflow from S-01.18 for the correct Node version to use.

3. `permissions: contents: write` is required for `peaceiris/actions-gh-pages` to push to the `gh-pages` branch.

4. The `GITHUB_TOKEN` secret is automatically available in all GitHub Actions — no manual secret setup required.

5. This workflow is separate from `ci.yml` (which runs on PRs). Do not add a `pull_request` trigger to this file — deployment should only happen on merges to `main`.

6. In GitHub repository Settings → Pages → Source, select "Deploy from a branch" and choose the `gh-pages` branch, root `/`. This is a one-time manual step documented in S-15.03.

## Done When

- [ ] `.github/workflows/deploy.yml` exists
- [ ] Triggers only on push to `main`
- [ ] Runs `npm ci` → `npm run build` → deploys `dist/`
- [ ] Uses `peaceiris/actions-gh-pages@v3` (pinned version)
- [ ] `permissions: contents: write` set
