# S-07.01: Phase Mode Prop

**Epic:** E-07
**Size:** XS
**Dependencies:** S-06 (BlockEditor exists and renders a Blockly workspace)

## Skills to Load

- `react-component-structure` — prop interface conventions, named exports
- `typescript-standards` — union literal types for constrained prop values
- `naming-conventions` — prop naming, file locations

## What

Add a `phase: 1 | 2 | 3` prop to `BlockEditor.tsx`. The prop controls rendering behavior: Phase 1 is default block-only rendering; Phase 2 enables syntax-visible mode (implemented in S-07.02); Phase 3 renders blocks in read-only mode (consumed by E-08's `EditorToggle`).

This story is purely additive — it wires the prop and passes it into the relevant sub-components via context or direct prop-drilling. No visible change in Phase 1 behaviour after this story.

## Tasks

1. Update `BlockEditor.tsx` props interface:
   ```ts
   export interface BlockEditorProps {
     phase: 1 | 2 | 3;
     /** existing props remain unchanged */
   }
   ```

2. Pass `phase` down to the workspace configuration function. When `phase === 3`, set the Blockly workspace option `readOnly: true`. When `phase === 1` or `phase === 2`, `readOnly` remains `false`.

3. Document the prop at the call site in the lesson runner (wherever `<BlockEditor />` is rendered). Add a comment explaining the contract:
   ```tsx
   {/* phase 1: blocks only, phase 2: blocks + syntax labels, phase 3: read-only reference */}
   <BlockEditor phase={currentPhase} ... />
   ```

4. Confirm `npm run lint` and `npx tsc --noEmit` pass — the prop must be required (no default), forcing all call sites to be explicit about phase.

## Done When

- [ ] `BlockEditorProps` includes `phase: 1 | 2 | 3` (required, not optional)
- [ ] Phase 3 sets Blockly workspace to `readOnly: true`
- [ ] Phase 1 and 2 leave the workspace editable
- [ ] All existing call sites updated to pass `phase`
- [ ] `npm run lint` and `npx tsc --noEmit` pass with no new errors
