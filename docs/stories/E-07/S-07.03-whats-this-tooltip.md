# S-07.03: "What's This?" Tooltip

**Epic:** E-07
**Size:** XS
**Dependencies:** S-07.02 (code labels visible in Phase 2)

## Skills to Load

- `accessibility` — keyboard focusable trigger, `role="tooltip"`, `aria-describedby`
- `react-component-structure` — small self-contained component, co-located with `BlockEditor`
- `formatting` — Prettier-compliant JSX

## What

In Phase 2, a small "?" icon button appears near the block editor (top-right corner of the editor container). Clicking or focusing the button reveals a tooltip/popover with the text:

> "These are the actual code commands your blocks run! In the next level, you'll type them yourself."

The tooltip is purely informational — it does not affect game state. It is only rendered when `phase === 2`.

## Tasks

1. Create `src/editor/CodeLabelTooltip.tsx`:
   ```tsx
   export function CodeLabelTooltip(): React.ReactElement {
     const [open, setOpen] = useState(false);
     const id = 'code-label-tooltip';
     return (
       <div className="relative inline-block">
         <button
           type="button"
           aria-label="What are these code labels?"
           aria-describedby={open ? id : undefined}
           onClick={() => setOpen((prev) => !prev)}
           onBlur={() => setOpen(false)}
           className="..." // pixel-art styled "?" button
         >
           ?
         </button>
         {open && (
           <div
             id={id}
             role="tooltip"
             className="..." // popover styles
           >
             These are the actual code commands your blocks run! In the next level,
             you&apos;ll type them yourself.
           </div>
         )}
       </div>
     );
   }
   ```

2. Render `<CodeLabelTooltip />` inside `BlockEditor.tsx` conditionally: only when `phase === 2`. Position it at the top-right of the editor container using absolute or flex layout.

3. The tooltip must be keyboard-accessible:
   - The "?" button receives focus via `Tab`
   - `Enter` or `Space` opens and closes it
   - `Escape` closes it (add `onKeyDown` handler for `Escape`)
   - When open, the tooltip text is announced by screen readers via `role="tooltip"` and `aria-describedby`

4. Tooltip appearance: small pixel-art styled card, max-width `260px`, positioned above or below the "?" button depending on available space (use a fixed direction for v1; responsive repositioning is a future enhancement).
