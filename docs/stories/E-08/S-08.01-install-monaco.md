# S-08.01: Install Monaco Editor + Skeleton Component

**Epic:** E-08
**Size:** S
**Dependencies:** E-06 complete (editor/ directory exists and has established patterns)

## Skills to Load

- `testing-strategy` — background on the outcome-focused validation philosophy; this story adds no executable code, so no new tests are required
- `dependency-policy` — vetting new dependencies before install
- `react-component-structure` — skeleton file layout, named exports
- `typescript-standards` — types for Monaco loader, no `any`
- `anti-patterns` — do not eagerly import Monaco at the module level

## What

Install `@monaco-editor/react` and create a lazy-loaded skeleton component at `src/editor/SyntaxEditor.tsx`. The skeleton renders Monaco inside a `<div>` with placeholder configuration. Full implementation follows in S-08.02.

Monaco is approximately 2 MB of JavaScript. It must never be part of the initial bundle. Use React's `lazy` + `Suspense` at the call site so Monaco is only fetched when a Phase 3 lesson is entered for the first time.

## Tasks

1. Install the package:
   ```bash
   npm install @monaco-editor/react
   ```
   `@monaco-editor/react` includes its own TypeScript types — no separate `@types/` package needed. Confirm this after install by checking that `import Editor from '@monaco-editor/react'` resolves without error in `npx tsc --noEmit`.

2. Create `src/editor/SyntaxEditor.tsx` as a skeleton:
   ```tsx
   import Editor from '@monaco-editor/react';

   export interface SyntaxEditorProps {
     value: string;
     onChange: (code: string) => void;
     readOnly?: boolean;
     availableFunctions: string[];
   }

   export function SyntaxEditor({ value, onChange, readOnly = false, availableFunctions }: SyntaxEditorProps): React.ReactElement {
     // Full implementation in S-08.02
     // availableFunctions used in S-08.04
     void availableFunctions;
     return (
       <div style={{ height: '400px' }}>
         <Editor
           language="javascript"
           value={value}
           onChange={(v) => onChange(v ?? '')}
           options={{ readOnly }}
         />
       </div>
     );
   }
   ```

3. Document lazy loading at the import site (wherever `SyntaxEditor` is consumed — likely `EditorToggle.tsx` in S-08.03, or the lesson runner layout):
   ```tsx
   const SyntaxEditor = React.lazy(() =>
     import('./SyntaxEditor').then((m) => ({ default: m.SyntaxEditor }))
   );
   // Wrap usage in <Suspense fallback={<LoadingSpinner />}>
   ```
   Add this comment pattern to `SyntaxEditor.tsx` itself as a usage note in the JSDoc.

4. Verify the Vite build produces a separate chunk for Monaco (check `dist/` after `npm run build` — look for a large `*.js` file that is not the main bundle).

5. Run `npm run lint` and `npx tsc --noEmit` to confirm no type errors introduced.
