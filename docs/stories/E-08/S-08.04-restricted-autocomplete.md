# S-08.04: Restrict Autocomplete to availableFunctions

**Epic:** E-08
**Size:** S
**Dependencies:** S-08.02 (`SyntaxEditor` implemented with editor instance ref)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `unit-tests` — Vitest tests for the pure logic split out of the component
- `component-tests` — Testing Library behavior tests for the component
- `typescript-standards` — typed Monaco API, no `any`; use `import type` for Monaco types
- `security-rules` — autocomplete restriction is a UX layer, not a security boundary (the interpreter/sandbox handles security); do not conflate the two
- `content-pack-system` — `availableFunctions` comes from `lesson.json`; each entry has `name`, `signature`, `description`, `example`

## What

Register a Monaco completion item provider that suggests only the lesson's `availableFunctions`. Each suggestion shows the function signature and description from the content pack. The built-in JavaScript IntelliSense (which would suggest DOM APIs, `window`, `console`, etc.) must be suppressed for the `javascript` language in this editor instance.

This is purely a UX improvement — restricting autocomplete helps younger players discover available functions without being overwhelmed by irrelevant suggestions. The sandbox (E-09) handles actual security enforcement.

## Tasks

1. Suppress Monaco's built-in JavaScript language features by configuring the language defaults before mounting:
   ```ts
   import * as monaco from 'monaco-editor';

   // Call once, before the Editor component mounts
   monaco.languages.typescript.javascriptDefaults.setDiagnosticsOptions({
     noSemanticValidation: true,
     noSyntaxValidation: false,   // keep syntax squiggles
   });
   monaco.languages.typescript.javascriptDefaults.setCompilerOptions({
     noLib: true,                 // disables DOM lib type suggestions
     allowNonTsExtensions: true,
   });
   ```

2. In `SyntaxEditor.tsx`, use `useEffect` to register the completion provider after the editor mounts. Use the editor instance ref from S-08.02:
   ```ts
   useEffect(() => {
     const disposable = monaco.languages.registerCompletionItemProvider('javascript', {
       provideCompletionItems(model, position) {
         const word = model.getWordUntilPosition(position);
         const range: monaco.IRange = {
           startLineNumber: position.lineNumber,
           endLineNumber: position.lineNumber,
           startColumn: word.startColumn,
           endColumn: word.endColumn,
         };
         const suggestions: monaco.languages.CompletionItem[] = availableFunctions.map((fn) => ({
           label: fn.name,
           kind: monaco.languages.CompletionItemKind.Function,
           detail: fn.signature,
           documentation: fn.description,
           insertText: fn.name + '()',
           range,
         }));
         return { suggestions };
       },
     });
     return () => disposable.dispose();
   }, [availableFunctions]);
   ```

3. The `availableFunctions` prop type must be updated to match the full shape from the content pack:
   ```ts
   export interface AvailableFunction {
     name: string;
     signature: string;
     description: string;
     example: string;
   }
   // Update SyntaxEditorProps:
   availableFunctions: AvailableFunction[];
   ```
   Update all call sites accordingly.

4. Verify in the browser: typing `fill` should suggest `fillBackground` with its signature and description. Typing `window` should produce no suggestions. Built-in JS globals (`console`, `Math`, etc.) should not appear.
