## What

Replace the skeleton `SyntaxEditor.tsx` with the full implementation. Configure Monaco with the correct language, theme, font, and options. The component is controlled: `value` and `onChange` are the source of truth ŌĆö the parent (lesson runner) owns the code string.

Press Start 2P is not suitable here ŌĆö it is too wide for a syntax editor and will make code unreadable. Use `Fira Code` (a programming font with ligatures, available on Google Fonts) at `14px` as the editor font. Add `Fira Code` to the Google Fonts import in `index.html` if not already present.

## Tasks

1. Update the Monaco `Editor` options object:
   ```ts
   const EDITOR_OPTIONS: editor.IStandaloneEditorConstructionOptions = {
     language: 'javascript',
     theme: 'vs-dark',
     fontFamily: '"Fira Code", monospace',
     fontSize: 14,
     fontLigatures: true,
     lineNumbers: 'on',
     minimap: { enabled: false },
     scrollBeyondLastLine: false,
     wordWrap: 'on',
     tabSize: 2,
     automaticLayout: true,   // resizes when container changes size
     readOnly: false,          // overridden by prop
   };
   ```

2. The full component signature:
   ```tsx
   export interface SyntaxEditorProps {
     value: string;
     onChange: (code: string) => void;
     readOnly?: boolean;
     availableFunctions: string[];  // used by S-08.04 completion provider
   }
   ```

3. Wire `onChange`: Monaco's `onChange` fires with `string | undefined`; always coerce `undefined` to `''` before calling the prop callback.

4. Store the Monaco editor instance in a `ref` typed as `editor.IStandaloneCodeEditor | null`. This ref will be needed in S-08.04 for registering the completion provider.

5. Add `Fira Code` to `index.html`:
   ```html
   <link
     href="https://fonts.googleapis.com/css2?family=Fira+Code:wght@400;500&display=swap"
     rel="stylesheet"
   />
   ```

6. Set container height via a prop (`height?: string`, defaulting to `'400px'`) so the lesson runner can control editor size.
