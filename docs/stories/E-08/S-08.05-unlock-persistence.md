# S-08.05: Syntax Editor Unlock Persistence

**Epic:** E-08
**Size:** XS
**Dependencies:** S-08.03 (`EditorToggle` exists), E-12 complete (`progressStore` with per-lesson schema)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `unit-tests` — Vitest AAA+O pattern for the pure logic added here
- `localstorage-pattern` — read/write via `progressStore`, never direct `localStorage` calls
- `typescript-standards` — explicit return types, no `any`
- `naming-conventions` — `syntaxUnlocked` field is defined in the E-12 progress schema

## What

When Phase 2 is completed, the lesson runner writes `syntaxUnlocked: true` to the lesson's progress entry in `progressStore`. On Phase 3 load, the lesson runner reads this flag. If `true`, the `EditorToggle` (with the `SyntaxEditor` as default view) is rendered. This ensures "once unlocked, always unlocked for that lesson" — the player can return to a completed Phase 3 lesson and the syntax editor is still there.

This story is the wiring layer — it connects the phase transition event (from E-11's lesson runner) to the persistence layer (E-12's `progressStore`) and the UI gate (E-08's `EditorToggle`).

## Tasks

1. In the lesson runner (E-11), after Phase 2 success is confirmed, call:
   ```ts
   progressStore.updateLesson(packId, lessonId, { syntaxUnlocked: true });
   ```

2. In the lesson runner's Phase 3 initialization, read the flag:
   ```ts
   const progress = progressStore.loadLesson(packId, lessonId);
   const syntaxUnlocked = progress?.syntaxUnlocked ?? false;
   ```

3. Gate the `EditorToggle` on this flag: only render `<EditorToggle />` when `syntaxUnlocked === true`. If somehow Phase 3 is reached but `syntaxUnlocked` is `false` (should not happen in normal flow, but guard defensively), fall back to `<BlockEditor phase={3} />` with a warning logged via `console.warn`.

4. Confirm the flag persists across page refresh: complete Phase 2, refresh the browser, navigate back to Phase 3 of that lesson — the syntax editor must be present.
