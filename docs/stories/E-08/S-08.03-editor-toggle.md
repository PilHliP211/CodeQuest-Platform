# S-08.03: EditorToggle Tab Bar Component

**Epic:** E-08
**Size:** S
**Dependencies:** S-08.02 (`SyntaxEditor` implemented), S-07.01 (`BlockEditor` phase prop)

## Skills to Load

- `react-component-structure` — controlled tab component, named exports
- `accessibility` — ARIA tab pattern (`role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls`)
- `typescript-standards` — union literal type for `activeView`
- `formatting` — Prettier-compliant JSX

## What

`EditorToggle.tsx` is the tab bar that hosts both editors in Phase 3. It renders two tabs — "Blocks" and "Code" — and the content area below shows either `<BlockEditor phase={3} />` (always read-only in Phase 3, per E-07 S-07.01) or `<SyntaxEditor />`. The parent controls the active view via props; `EditorToggle` is a controlled component.

In Phase 3, the "Code" tab is the default/primary view. The "Blocks" tab label should include a visual hint ("read-only") so the player knows they cannot run from the blocks view.

## Tasks

1. Create `src/editor/EditorToggle.tsx`:
   ```tsx
   export type EditorView = 'blocks' | 'syntax';

   export interface EditorToggleProps {
     activeView: EditorView;
     onViewChange: (view: EditorView) => void;
     // Props forwarded to the active editor
     syntaxEditorProps: SyntaxEditorProps;
     blockEditorProps: Omit<BlockEditorProps, 'phase'>;  // phase is always 3 here
   }
   ```

2. Render an ARIA tab bar:
   ```tsx
   <div role="tablist" aria-label="Editor view">
     <button
       role="tab"
       aria-selected={activeView === 'syntax'}
       aria-controls="syntax-panel"
       id="tab-syntax"
       onClick={() => onViewChange('syntax')}
     >
       Code
     </button>
     <button
       role="tab"
       aria-selected={activeView === 'blocks'}
       aria-controls="blocks-panel"
       id="tab-blocks"
       onClick={() => onViewChange('blocks')}
     >
       Blocks <span aria-label="(read-only)">(view only)</span>
     </button>
   </div>
   ```

3. Render the active panel:
   ```tsx
   <div id="syntax-panel" role="tabpanel" aria-labelledby="tab-syntax" hidden={activeView !== 'syntax'}>
     <React.Suspense fallback={<LoadingSpinner />}>
       <SyntaxEditor {...syntaxEditorProps} />
     </React.Suspense>
   </div>
   <div id="blocks-panel" role="tabpanel" aria-labelledby="tab-blocks" hidden={activeView !== 'blocks'}>
     <BlockEditor phase={3} {...blockEditorProps} />
   </div>
   ```
   Using `hidden` (not conditional rendering) keeps both panels mounted, which prevents Monaco from re-initializing on tab switch.

4. Style the active tab with a distinct highlight (border-bottom or background color) using Tailwind classes consistent with the pixel-art theme.

5. Keyboard navigation: left/right arrow keys move focus between tabs (standard ARIA tab pattern). Implement with `onKeyDown` on the tablist.
