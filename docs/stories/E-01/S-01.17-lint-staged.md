# S-01.17: Configure lint-staged

**Epic:** E-01
**Size:** XS
**Dependencies:** S-01.09 (ESLint rules), S-01.11 (Prettier compat), S-01.16 (Husky)

## Skills to Load

- `ci-and-quality-gates` — pre-commit gate semantics, especially `--max-warnings=0`
- `eslint-rules` — to understand what the hook is enforcing
- `formatting` — same for Prettier

## What

Wire `lint-staged` into the pre-commit hook so staged files are linted and format-checked before every commit. Warnings are commit-blocking (`--max-warnings=0`).

## Tasks

1. Add `lint-staged` config to `package.json`:
   ```json
   "lint-staged": {
     "src/**/*.{ts,tsx}": [
       "eslint --max-warnings=0",
       "prettier --check"
     ],
     "src/**/*.{json,css}": [
       "prettier --check"
     ]
   }
   ```
2. Replace `.husky/pre-commit` contents with:
   ```sh
   npx lint-staged
   ```
3. Verify by:
   - Adding a temporary `console.log("test")` to a TS file
   - `git add` it
   - `git commit -m "test"` — should be rejected by ESLint
   - Remove the offending line and confirm the commit succeeds

## Done When

- [ ] `lint-staged` config in `package.json`
- [ ] `.husky/pre-commit` runs `npx lint-staged`
- [ ] Committing a file with a lint error fails
- [ ] Committing a file with a format issue fails
- [ ] Clean files commit successfully
