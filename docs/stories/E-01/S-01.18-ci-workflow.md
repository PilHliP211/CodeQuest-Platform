# S-01.18: GitHub Actions CI Workflow

**Epic:** E-01
**Size:** S
**Dependencies:** All script-creating stories (S-01.07, S-01.10, S-01.12, etc.)

## Skills to Load

- `ci-and-quality-gates` — the source of truth for what the workflow must run

## What

A CI workflow that runs the four quality gates on every PR and push to `main`. Any failure blocks the merge.

## Tasks

1. Create `.github/workflows/ci.yml`:
   ```yaml
   name: CI

   on:
     push:
       branches: [main]
     pull_request:
       branches: [main]

   jobs:
     check:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: 20
             cache: npm
         - run: npm ci
         - run: npm run lint
         - run: npm run typecheck
         - run: npm run format:check
         - run: npm run build
   ```
2. Validate the YAML locally (any YAML linter or just visual check)
3. Confirm all five `npm run` commands exist in `package.json`
4. After merging this story, the next PR will trigger the workflow — verify it runs and passes

## Done When

- [x] `.github/workflows/ci.yml` exists and is valid YAML
- [x] Workflow steps: `lint`, `typecheck`, `format:check`, `build`
- [x] Workflow runs on push to `main` and on PRs
- [x] All referenced npm scripts exist

## Notes

The CI pipeline is the single source of truth for "is this code shippable?" Every check in CI must also be runnable locally. No CI-only magic. See the shared AI guidance in `.claude/skills/ci-and-quality-gates.md`.
