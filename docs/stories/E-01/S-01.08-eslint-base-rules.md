# S-01.08: ESLint Base + strictTypeChecked Rules

**Epic:** E-01
**Size:** S
**Dependencies:** S-01.07

## Skills to Load

- `eslint-rules` — the rule rationale and the `--max-warnings=0` policy
- `ci-and-quality-gates` — how lint fits in the four-gate pipeline

## What

Create `eslint.config.js` (flat config) using the `strictTypeChecked` and `stylisticTypeChecked` configs from `typescript-eslint`. This is the baseline.

## Tasks

1. Create `eslint.config.js`:
   ```js
   import js from '@eslint/js';
   import tseslint from 'typescript-eslint';
   import reactHooks from 'eslint-plugin-react-hooks';
   import reactRefresh from 'eslint-plugin-react-refresh';

   export default tseslint.config(
     { ignores: ['dist/', 'node_modules/'] },
     js.configs.recommended,
     ...tseslint.configs.strictTypeChecked,
     ...tseslint.configs.stylisticTypeChecked,
     {
       languageOptions: {
         parserOptions: {
           projectService: true,
           tsconfigRootDir: import.meta.dirname,
         },
       },
     },
     {
       plugins: {
         'react-hooks': reactHooks,
         'react-refresh': reactRefresh,
       },
       rules: {
         ...reactHooks.configs.recommended.rules,
         'react-refresh/only-export-components': ['warn', { allowConstantExport: true }],
       },
     },
   );
   ```
2. Run `npm run lint` and resolve any errors in the scaffold
3. Project-specific rule overrides come in S-01.09

## Done When

- [ ] `eslint.config.js` exists and uses flat config format
- [ ] `strictTypeChecked` and `stylisticTypeChecked` configs are applied
- [ ] React hooks plugin rules are active
- [ ] `npm run lint` exits 0 on the scaffold
