# S-01.09: ESLint Project-Specific Rule Overrides

**Epic:** E-01
**Size:** S
**Dependencies:** S-01.08

## Skills to Load

- `eslint-rules` — full project rule rationale
- `typescript-standards` — context for the type-aware rules being added
- `anti-patterns` — the rules in this story enforce these anti-patterns

## What

Add the project-specific rules on top of the strict baseline. These enforce CodeQuest's hard rules.

## Tasks

1. Add to the main rules block in `eslint.config.js`:
   ```js
   rules: {
     // No silent debug logs
     'no-console': ['error', { allow: ['warn', 'error'] }],

     // Type-aware additions
     '@typescript-eslint/no-unused-vars': ['error', {
       argsIgnorePattern: '^_',
       varsIgnorePattern: '^_',
     }],
     '@typescript-eslint/explicit-function-return-type': ['error', {
       allowExpressions: true,
       allowTypedFunctionExpressions: true,
     }],
     '@typescript-eslint/no-floating-promises': 'error',
     '@typescript-eslint/no-misused-promises': 'error',
     '@typescript-eslint/strict-boolean-expressions': 'error',
     '@typescript-eslint/switch-exhaustiveness-check': 'error',

     // General quality
     'eqeqeq': ['error', 'always'],
     'no-var': 'error',
     'prefer-const': 'error',
     'curly': ['error', 'all'],
   },
   ```
2. Run `npm run lint` and confirm it still passes
3. Optionally add a sample violation, run lint to confirm it's caught, then revert

## Done When

- [ ] All listed rules are active
- [ ] `npm run lint` passes
- [ ] Adding `console.log("test")` to `App.tsx` causes a lint error
- [ ] See `.claude/skills/eslint-rules.md` for the rule rationale
