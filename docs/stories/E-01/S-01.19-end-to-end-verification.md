# S-01.19: End-to-End Gate Verification

**Epic:** E-01
**Size:** XS
**Dependencies:** All other E-01 stories

## Skills to Load

- `ci-and-quality-gates` — the gates being verified
- `anti-patterns` — last-chance check before declaring E-01 done

## What

A final verification that every quality gate works end-to-end on a fresh clone. This is the "ship it" check for E-01.

## Tasks

1. From a clean state (`rm -rf node_modules dist`), run:
   ```
   npm install
   npm run lint
   npm run typecheck
   npm run format:check
   npm run build
   npm run dev   # then Ctrl+C
   ```
2. All commands must exit 0 (or start successfully for `dev`)
3. Test the pre-commit hook:
   - Add a temporary lint violation, attempt commit, confirm rejection
   - Add a temporary format violation, attempt commit, confirm rejection
   - Revert and confirm clean commit succeeds
4. Push the branch and verify the GitHub Actions CI workflow runs and passes
5. Document any fresh-clone gotchas in `README.md` if you find them

## Done When

- [ ] All four gates pass on a fresh clone
- [ ] `npm run dev` starts cleanly
- [ ] `npm run preview` serves the production build
- [ ] Pre-commit hook rejects bad commits
- [ ] CI workflow runs and passes on GitHub
- [ ] No "you have to do X manually first" tribal knowledge — everything is captured in scripts or README
