# S-01.02: Strict tsconfig.json

**Epic:** E-01
**Size:** XS
**Dependencies:** S-01.01

## Skills to Load

- `typescript-standards` — the canonical reference for which strict flags and why
- `ci-and-quality-gates` — to understand the role of `tsc --noEmit` in the pipeline

## What

Configure `tsconfig.json` for maximum type safety. Every strict flag enabled.

## Tasks

1. Update `tsconfig.json` `compilerOptions`:
   ```jsonc
   {
     "strict": true,
     "noUncheckedIndexedAccess": true,
     "exactOptionalPropertyTypes": true,
     "noImplicitReturns": true,
     "noFallthroughCasesInSwitch": true,
     "noUnusedLocals": true,
     "noUnusedParameters": true,
     "forceConsistentCasingInFileNames": true,
     "isolatedModules": true,
     "moduleResolution": "bundler",
     "jsx": "react-jsx",
     "target": "ES2022",
     "lib": ["ES2022", "DOM", "DOM.Iterable"]
   }
   ```
2. Run `npx tsc --noEmit` and resolve any errors that surface in the existing code (likely none — the scaffold is minimal)
3. Add a `typecheck` script to `package.json`: `"typecheck": "tsc --noEmit"`

## Done When

- [x] `npx tsc --noEmit` exits 0
- [x] All listed strict flags are present
- [x] `npm run typecheck` works as a shortcut
- [x] Vite dev server still runs

## Notes

- `noUncheckedIndexedAccess` is intentionally strict — `array[i]` becomes `T | undefined`. This prevents an entire class of runtime errors.
- `exactOptionalPropertyTypes` means `{ name?: string }` does NOT accept `{ name: undefined }`. Catches real bugs.
- See `.claude/skills/typescript-standards.md` for the rationale.
