# S-01.15: Vite Base Path for GitHub Pages

**Epic:** E-01
**Size:** XS
**Dependencies:** S-01.01

## Skills to Load

- `architecture-overview` — context for why we deploy under a subpath

## What

Configure Vite's `base` so the production build works when served from `/codequest-platform/`.

## Tasks

1. Update `vite.config.ts`:
   ```ts
   export default defineConfig({
     plugins: [react(), tailwindcss()],
     base: '/codequest-platform/',
     // ... other config
   });
   ```
2. Run `npm run build`
3. Inspect `dist/index.html` — asset paths should start with `/codequest-platform/assets/...`
4. Run `npm run preview` and verify the app loads under the subpath

## Done When

- [ ] `base: '/codequest-platform/'` is set
- [ ] Built `index.html` references `/codequest-platform/assets/...`
- [ ] `npm run preview` serves the app correctly
