# S-01.06: Path Aliases

**Epic:** E-01
**Size:** XS
**Dependencies:** S-01.02 (tsconfig), S-01.05 (folders)

## Skills to Load

- `architecture-overview` — to confirm `@/` and `@content/` map to the intended roots

## What

Configure `@/` to resolve to `src/` and `@content/` to resolve to `content/`. Both TypeScript and Vite need to know.

## Tasks

1. Add to `tsconfig.json` `compilerOptions`:
   ```jsonc
   {
     "baseUrl": ".",
     "paths": {
       "@/*": ["src/*"],
       "@content/*": ["content/*"]
     }
   }
   ```
2. Add to `vite.config.ts`:
   ```ts
   resolve: {
     alias: {
       '@': path.resolve(__dirname, 'src'),
       '@content': path.resolve(__dirname, 'content'),
     },
   }
   ```
3. Create a sample import in `App.tsx` to confirm both aliases resolve (you can remove it after verifying)
4. Run `npx tsc --noEmit` and `npm run dev` to confirm both build systems agree

## Done When

- [ ] `import x from '@/...'` resolves in both TypeScript and Vite
- [ ] `import y from '@content/...'` resolves in both TypeScript and Vite
- [ ] `npm run typecheck` passes
- [ ] `npm run build` succeeds
