# S-01.11: ESLint–Prettier Compatibility

**Epic:** E-01
**Size:** XS
**Dependencies:** S-01.09, S-01.10

## Skills to Load

- `eslint-rules` — to understand which rules are being disabled
- `formatting` — confirms ESLint must never fight Prettier

## What

Add `eslint-config-prettier` so ESLint stops trying to enforce formatting rules that Prettier owns. Without this, the two tools can fight.

## Tasks

1. Install: `npm install -D eslint-config-prettier`
2. In `eslint.config.js`, append the Prettier compat config as the LAST entry (it must come after all rule-defining configs to override them):
   ```js
   import eslintConfigPrettier from 'eslint-config-prettier';

   export default tseslint.config(
     // ... existing config ...
     eslintConfigPrettier, // must be last
   );
   ```
3. Run both `npm run lint` and `npm run format:check` — both should pass with no overlapping complaints

## Done When

- [ ] `eslint-config-prettier` installed
- [ ] Compat config is the last entry in `eslint.config.js`
- [ ] `npm run lint` passes
- [ ] `npm run format:check` passes
- [ ] No conflicting rules between ESLint and Prettier
