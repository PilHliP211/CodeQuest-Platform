## What

Wrap the React app in `<ContentProvider>` inside `main.tsx`, so `ContentContext` and `ContentErrorContext` are available to the entire component tree. This is a one-line wiring task ŌĆö the provider itself was built in S-03.04.

The second half of this story is the **swap test**: the PRD requires that switching content packs produces a different course with zero changes to platform code. Verify this manually and document the result in the Done When checklist.

## Tasks

1. Update `src/main.tsx` to add `ContentProvider` around `<App />`:

   ```tsx
   import { StrictMode } from 'react';
   import { createRoot } from 'react-dom/client';
   import { ProfileProvider } from '@/engine/ProfileContext';
   import { ContentProvider } from '@/engine/ContentContext';
   import { App } from './App';
   import './index.css';

   const root = document.getElementById('root');
   if (root === null) throw new Error('Root element #root not found in index.html');

   createRoot(root).render(
     <StrictMode>
       <ProfileProvider>
         <ContentProvider>
           <App />
         </ContentProvider>
       </ProfileProvider>
     </StrictMode>,
   );
   ```

   Provider nesting: `ProfileProvider` is outer, `ContentProvider` is inner. This order is intentional ŌĆö content does not depend on profile, but if a future feature ties content selection to the learner's profile, the profile will already be available.

2. Perform the swap test:
   - Create a minimal alternate content pack at `content/test-pack/course.json` with a different `id` ("test-pack") and `title` ("Test Course") but the same structural shape.
   - Temporarily change the import in `contentLoader.ts` from `../../content/flag-hunter/course.json` to `../../content/test-pack/course.json`.
   - Run `npm run dev` and verify the app loads with the new course title visible (via a temporary `console.warn` or the browser React DevTools inspection of ContentContext).
   - Revert both the import and delete the test pack.
   - No changes to `App.tsx`, `ContentContext.tsx`, or any other platform file should be needed.

3. After reverting, confirm all four quality gates still pass.
