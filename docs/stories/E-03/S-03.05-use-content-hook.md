# S-03.05: useContent Hook

**Epic:** E-03
**Size:** XS
**Dependencies:** S-03.04 (ContentContext)

## Skills to Load

- `react-context-pattern` — hook-with-provider-check, identical pattern to `useProfile` from S-02.06

## What

The `useContent` hook is the public API for reading the loaded `Course`. It reads from `ContentContext` and throws if called outside the provider or when the course failed to load. This is the exact same pattern as `useProfile` from E-02.

Components never import `ContentContext` or call `useContext(ContentContext)` directly — they always go through `useContent`. This keeps the context shape an internal implementation detail.

## Tasks

1. Add `useContent` to `src/engine/ContentContext.tsx` (at the bottom of the file, below the provider):

   ```ts
   import { useContext } from 'react';
   import { ContentContext, type ContentContextValue } from './ContentContext';

   export function useContent(): ContentContextValue {
     const ctx = useContext(ContentContext);
     if (ctx === undefined) {
       throw new Error(
         'useContent must be used within <ContentProvider> and requires a successfully loaded content pack.',
       );
     }
     return ctx;
   }
   ```

2. The error thrown here will surface as a React render error if a component calls `useContent()` when the course failed to load. That case is prevented by the error gate in S-03.06, which renders a full-screen error UI instead of the child tree when `ContentContext` is `undefined`.

3. Do not export `ContentContext` for direct `useContext` use — `useContent` is the only entry point.

## Done When

- [ ] `useContent()` returns `{ course: Course }` when used inside a successfully loaded provider
- [ ] Calling `useContent()` outside the provider throws an `Error` with a clear message
- [ ] Hook has an explicit return type (`ContentContextValue`)
- [ ] `npx tsc --noEmit` and `npm run lint` pass
