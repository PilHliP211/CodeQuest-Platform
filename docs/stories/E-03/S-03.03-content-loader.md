## What

`contentLoader.ts` is the single place the platform imports content pack data. In v1, this is a static Vite import of `content/flag-hunter/course.json`. The file runs the `isCourse` type guard on the imported JSON and either returns a typed `Course` or throws a descriptive `Error` that explains exactly what went wrong and where.

This is the only file in the platform that knows the literal import path `content/flag-hunter/course.json`. When E-17 (multi-pack support) arrives, this file ŌĆö and only this file ŌĆö will change. All other engine code talks to the loaded `Course` object, never to raw JSON.

The function is called once at startup (inside `ContentContext`), not on every render.

## Tasks

1. Create `src/engine/contentLoader.ts`:

   ```ts
   import type { Course } from '@/types/content';
   import { isCourse } from './contentGuard';
   import rawCourse from '../../content/flag-hunter/course.json';

   /**
    * Load and validate the active content pack.
    *
    * In v1 this is a static import bundled at build time.
    * In v2+ this will be replaced with a dynamic import or fetch.
    *
    * @throws Error if the content pack fails validation
    */
   export function loadContent(): Course {
     if (!isCourse(rawCourse)) {
       throw new Error(
         'Content pack validation failed: "content/flag-hunter/course.json" does not ' +
           'match the required Course schema. Check that all required fields are present ' +
           '(id, title, description, version, map, lessons, xp, villain, player).',
       );
     }
     return rawCourse;
   }
   ```

2. Add `"resolveJsonModule": true` to `tsconfig.json` if it isn't already set ŌĆö Vite handles JSON imports natively but `tsc` needs this flag to type-check them.

3. The `rawCourse` import will be typed as `unknown` by TypeScript when `resolveJsonModule` is true and the file is deep inside `content/`. Alternatively, it may be typed loosely as the inferred JSON shape. Either way, `isCourse` narrows it to `Course` before it is returned. Do not cast with `as Course` ŌĆö let the guard do the narrowing.

   If TypeScript infers the JSON as a concrete literal type (not `unknown`), the `isCourse` call will still compile correctly because `Course` is structurally compatible.

4. Do not call `loadContent()` at module load time (top level). Call it lazily from `ContentContext`. This keeps error boundaries working correctly with React.
