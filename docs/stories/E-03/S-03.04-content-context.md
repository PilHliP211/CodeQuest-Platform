## What

`ContentContext.tsx` provides the loaded `Course` to all components in the tree. The provider calls `loadContent()` once inside a `useState` initializer, catches any thrown `Error`, and stores either the `Course` or an error message in state.

This mirrors the shape of `ProfileContext` from E-02 exactly — same file layout, same `createContext` + named provider function pattern. The key difference: content never changes after load (no `update` actions), so the context value only exposes the data and an optional error string.

The error state is consumed by the content load error gate (S-03.06), not by this file. The provider itself is just the data layer.

## Tasks

1. Create `src/engine/ContentContext.tsx`:

   ```tsx
   import {
     createContext,
     useMemo,
     useState,
     type ReactNode,
   } from 'react';
   import type { Course } from '@/types/content';
   import { loadContent } from './contentLoader';

   export interface ContentContextValue {
     course: Course;
   }

   export const ContentContext = createContext<ContentContextValue | undefined>(undefined);

   interface ContentProviderProps {
     children: ReactNode;
   }

   interface ContentState {
     course: Course | null;
     error: string | null;
   }

   function loadInitialState(): ContentState {
     try {
       const course = loadContent();
       return { course, error: null };
     } catch (err) {
       const message =
         err instanceof Error
           ? err.message
           : 'An unknown error occurred while loading the content pack.';
       return { course: null, error: message };
     }
   }

   export function ContentProvider({ children }: ContentProviderProps): React.ReactElement {
     const [state] = useState<ContentState>(loadInitialState);

     const value = useMemo<ContentContextValue | undefined>(() => {
       if (state.course === null) return undefined;
       return { course: state.course };
     }, [state.course]);

     return (
       <ContentContext.Provider value={value}>
         {children}
       </ContentContext.Provider>
     );
   }

   /**
    * Exposed separately so the error gate can read the error without
    * going through the full context (which requires a valid course).
    */
   export function useContentLoadError(): string | null {
     const [state] = useState<ContentState>(loadInitialState);
     return state.error;
   }
   ```

   > Note on `useContentLoadError`: because `useState(loadInitialState)` memoizes the result after the first call in a given component, calling the same initializer in two different hooks will trigger it twice. The better pattern is to lift the state to a single location. Revise the implementation to keep a single `ContentState` at the provider level and expose the error through a second context or a prop, as described below.

2. Revised approach — hoist error state out of the hook into the provider and expose it via a separate context:

   ```tsx
   import {
     createContext,
     useContext,
     useMemo,
     useState,
     type ReactNode,
   } from 'react';
   import type { Course } from '@/types/content';
   import { loadContent } from './contentLoader';

   // --- Content context (only set when load succeeded) ---

   export interface ContentContextValue {
     course: Course;
   }

   export const ContentContext = createContext<ContentContextValue | undefined>(undefined);

   // --- Load error context (always set; null means no error) ---

   export const ContentErrorContext = createContext<string | null>(null);

   // --- Provider ---

   export function ContentProvider({ children }: { children: ReactNode }): React.ReactElement {
     const [contentState] = useState<{ course: Course | null; error: string | null }>(() => {
       try {
         return { course: loadContent(), error: null };
       } catch (err) {
         const message =
           err instanceof Error
             ? err.message
             : 'An unknown error occurred while loading the content pack.';
         return { course: null, error: message };
       }
     });

     const contentValue = useMemo<ContentContextValue | undefined>(
       () => (contentState.course !== null ? { course: contentState.course } : undefined),
       [contentState.course],
     );

     return (
       <ContentErrorContext.Provider value={contentState.error}>
         <ContentContext.Provider value={contentValue}>
           {children}
         </ContentContext.Provider>
       </ContentErrorContext.Provider>
     );
   }
   ```

3. Export `ContentContext` and `ContentErrorContext` from this file — but document that consumers should use the hooks (`useContent`, defined in S-03.05) rather than calling `useContext` directly.

4. State is initialized once via the `useState` initializer function. Content never changes after mount, so there are no setters.

## Done When

- [ ] `ContentContext.tsx` exports `ContentProvider`, `ContentContext`, and `ContentErrorContext`
- [ ] Provider calls `loadContent()` exactly once (in the `useState` initializer, not in the render body)
- [ ] If `loadContent()` throws, `contentState.error` is set to the error message; `course` is `null`
- [ ] If `loadContent()` succeeds, `contentState.error` is `null`; `course` is the typed `Course`
- [ ] `ContentContext.Provider` receives `undefined` when the course failed to load (so `useContent` will throw — handled by the gate in S-03.06)
- [ ] `ContentErrorContext.Provider` always wraps the tree so the error gate can read it even when the course is `null`
- [ ] Context value is memoized
- [ ] `npx tsc --noEmit` and `npm run lint` pass
