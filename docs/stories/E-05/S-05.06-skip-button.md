# S-05.06: Skip Button

**Epic:** E-05
**Size:** XS
**Dependencies:** S-05.05 (`NarrativePlayer` — skip button lives inside it)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `component-tests` — Testing Library behavior tests for the component added here
- `accessibility` — button must have a visible label and keyboard activation
- `react-component-structure` — inline element within an existing component, no new file needed

## What

A "Skip ▶▶" button positioned in the top-right corner of the `NarrativePlayer` container. It is only rendered when `skippable` is `true`. Clicking it calls `onComplete()` immediately, skipping all remaining scenes and lines.

This is not a separate file — it is the button element inside `NarrativePlayer.tsx`, already scaffolded in S-05.05. This story is about completing it correctly: accessible label, visual styling, and confirming the conditional render behavior.

## Tasks

1. In `NarrativePlayer.tsx`, confirm the skip button markup is complete:
   ```tsx
   {skippable && (
     <button
       type="button"
       className="absolute top-3 right-3 z-10 font-pixel text-xs text-white/70 hover:text-white focus:outline-none focus:ring-2 focus:ring-white/50 rounded px-2 py-1"
       aria-label="Skip narrative and continue"
       onClick={onComplete}
     >
       Skip ▶▶
     </button>
   )}
   ```
2. Requirements:
   - `type="button"` to prevent accidental form submission
   - `aria-label` that describes the action (not just "Skip") — use `"Skip narrative and continue"`
   - Visible focus ring (`focus:ring-2`) for keyboard navigation
   - Positioned with `absolute top-3 right-3 z-10` so it always floats above the scene content
   - Does not interfere with `DialogueBox` click-to-advance (z-index and pointer events are separate)
3. Manual test: tab to the button, press Enter — confirms keyboard activation works without advancing a dialogue line at the same time.
