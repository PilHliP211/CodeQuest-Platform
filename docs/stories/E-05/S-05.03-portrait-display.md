## What

Renders a character portrait image when a portrait path is provided, or renders nothing (empty fragment) when `portrait` is `null` or `undefined`. The image must be rendered with `image-rendering: pixelated` so pixel-art portraits don't blur when scaled up.

This is intentionally a small, focused component. It has no state and no logic beyond the null-check.

## Tasks

1. Create `src/components/Narrative/PortraitDisplay.tsx`:
   ```tsx
   export interface PortraitDisplayProps {
     portrait: string | null | undefined;
     alt: string;
   }

   export function PortraitDisplay({ portrait, alt }: PortraitDisplayProps): React.ReactElement {
     if (portrait == null) {
       return <></>;
     }
     return (
       <img
         src={portrait}
         alt={alt}
         className="w-16 h-16 object-contain"
         style={{ imageRendering: 'pixelated' }}
       />
     );
   }
   ```
2. The `alt` prop is required — callers must provide a meaningful description (e.g. `"Player portrait"`, `"Local Guide portrait"`). Never pass an empty string.
3. Sizing: default `w-16 h-16` (64×64px). The exact dimensions can be refined when real art assets are available — do not hardcode pixel values in inline styles.
4. `style={{ imageRendering: 'pixelated' }}` is needed because Tailwind does not have a utility for this CSS property in v3.
