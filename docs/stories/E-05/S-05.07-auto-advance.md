# S-05.07: Auto-Advance Mode

**Epic:** E-05
**Size:** XS
**Dependencies:** S-05.04 (`SceneRenderer` — `autoAdvanceMs` prop is already in its interface)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `component-tests` — Testing Library behavior tests for the component added here
- `react-component-structure` — `useEffect` + `setTimeout` cleanup pattern
- `typescript-standards` — `ReturnType<typeof setTimeout>` for the timer ref type

## What

When `autoAdvanceMs` is set on `SceneRenderer`, each dialogue line automatically advances to the next after that many milliseconds — without requiring a click. This enables timed cutscenes (e.g. a brief splash screen or a villain escape sequence) that don't need player input.

The timer must be cleared on component unmount and whenever the line index changes to avoid stale callbacks advancing the wrong line. The `DialogueBox` is still rendered (the player can still click to advance early), but the timer fires regardless.

## Tasks

1. In `SceneRenderer.tsx`, add the `useEffect` for auto-advance below the existing `handleAdvance` function:
   ```tsx
   useEffect(() => {
     if (autoAdvanceMs == null) return;

     const timerId: ReturnType<typeof setTimeout> = setTimeout(() => {
       handleAdvance();
     }, autoAdvanceMs);

     return () => {
       clearTimeout(timerId);
     };
   }, [lineIndex, autoAdvanceMs]); // handleAdvance is stable per render; lineIndex triggers reset
   ```
2. Important: `handleAdvance` captures `lineIndex` via closure. Since the effect depends on `lineIndex`, the timer resets each time the line changes — this is the correct behavior.
3. If `autoAdvanceMs` is set and the player clicks to advance early, the existing `handleAdvance` fires first, `lineIndex` changes, and the effect cleanup cancels the pending timer. No double-advance can occur.
4. There is no separate "auto-advance only" mode — the player can always click to advance early regardless of `autoAdvanceMs`. If a scene needs to block player input, set `scene.advanceOnInput = false` in the content JSON and conditionally hide or disable the `DialogueBox` click handler. That gating is a future content pack concern; this story only covers the timer.
