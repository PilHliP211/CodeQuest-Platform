# S-05.01: Narrative Type Definitions

**Epic:** E-05
**Size:** XS
**Dependencies:** S-03.01 (content types file exists at `src/types/content.ts`)

## Skills to Load

- `typescript-standards` — interface vs type, no `any`, ISO field conventions
- `naming-conventions` — types live in `src/types/content.ts`
- `architecture-overview` — confirms all content-shape types belong in `src/types/`

## What

Add `DialogueLine`, `NarrativeScene`, and `NarrativeScript` to the shared content types file. These types mirror the narrative script format documented in `CONTENT_SCHEMA.md` and are consumed by every component in `src/components/Narrative/`. Check whether any of these types were already partially defined in `src/types/content.ts` during E-03 — if so, extend rather than duplicate.

## Tasks

1. Open `src/types/content.ts` and add (or confirm already present):
   ```ts
   export interface DialogueLine {
     speaker: string;
     text: string;
     portrait?: string | null;
     emotion?: string;
   }

   export interface NarrativeScene {
     background?: string;
     dialogue: DialogueLine[];
     advanceOnInput?: boolean;
   }

   /** An ordered array of scenes that together form a complete narrative beat. */
   export type NarrativeScript = NarrativeScene[];
   ```
2. Confirm that `NarrativeScript` is referenced from the phase types already in `content.ts` (e.g. `phase1.intro.scenes` should be typed as `NarrativeScene[]`). Update those fields if they are typed as `unknown` or `object`.
3. Do not add validation logic here — that belongs in `contentLoader.ts` (E-03). This story is types only.

## Done When

- [ ] `DialogueLine`, `NarrativeScene`, and `NarrativeScript` are exported from `src/types/content.ts`
- [ ] All fields match the JSON schema in `CONTENT_SCHEMA.md` exactly (`portrait` is `string | null | undefined`, not just `string`)
- [ ] Phase intro/outro scene arrays in the content types now reference `NarrativeScene[]` instead of `unknown`
- [ ] `npx tsc --noEmit` passes
- [ ] `npm run lint` passes
