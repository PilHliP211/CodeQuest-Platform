# S-12.05: XP Total Display in HUD

**Epic:** E-12
**Size:** XS
**Dependencies:** S-12.02 (progressStore `getTotalXP`), S-02.11 (HUD component)

## Skills to Load

- `react-component-structure` — updating an existing component
- `localstorage-pattern` — reading a value on mount

## What

Update `HUD.tsx` (from E-02, S-02.11) to display the learner's total XP alongside their name. The XP is read from `progressStore.getTotalXP()` on component mount.

XP does not need to be reactive (live-updating during a lesson run) at this stage — the HUD re-mounts when the lesson completes and the map re-renders, which naturally refreshes the XP display.

## Tasks

1. Update `src/components/HUD/HUD.tsx`:
   ```tsx
   import { getTotalXP } from '@/engine/progressStore';

   export function HUD(): React.ReactElement {
     const { profile } = useProfile();
     const totalXP = getTotalXP();

     return (
       <header className="font-pixel flex items-center justify-between px-4 py-2 bg-gray-900 text-white">
         <span>{profile?.name ?? ''}</span>
         <span className="text-yellow-400">{totalXP} XP</span>
       </header>
     );
   }
   ```

2. `getTotalXP()` is a synchronous read — no `useEffect` or `useState` needed on mount for the initial render. If XP changes during a session (lesson completion), the HUD will reflect it on the next render triggered by the parent component tree.

3. If `totalXP` is 0 (first launch), display "0 XP" — do not hide the counter. Seeing "0 XP" motivates the player to earn some.

## Done When

- [ ] XP shows in yellow (`text-yellow-400` or equivalent)
- [ ] "0 XP" shown on first launch
