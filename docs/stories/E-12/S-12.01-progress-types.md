# S-12.01: Progress Type Definitions

**Epic:** E-12
**Size:** XS
**Dependencies:** S-03.01 (content types, for pack/lesson IDs)

## Skills to Load

- `typescript-standards` — interface definitions, no `any`
- `naming-conventions` — type file in `src/types/`

## What

Define the TypeScript types for all progress data. These types describe exactly what's stored in localStorage per lesson and in aggregate. They're the schema contract between the progress store and every consumer (lesson runner, map, HUD, settings).

## Tasks

1. Create `src/types/progress.ts`:
   ```ts
   /** Per-lesson progress stored under codequest:progress:{packId}:{lessonId} */
   export interface LessonProgress {
     phase1Complete: boolean;
     phase2Complete: boolean;
     phase3Complete: boolean;
     syntaxUnlocked: boolean;
     flagCollected: boolean;
     xpEarned: number;
     completedAt?: string; // ISO 8601 timestamp, set when phase3Complete becomes true
   }

   /** Per-course aggregate (keyed by lessonId) */
   export type CourseProgress = Record<string, LessonProgress>;

   /** Full profile progress (keyed by packId) */
   export type AllProgress = Record<string, CourseProgress>;
   ```

2. `LessonProgress` is the shape of each localStorage value. The key is `codequest:progress:{packId}:{lessonId}`.

3. XP total is stored separately under `codequest:xp` as a plain number string — not embedded in `LessonProgress`, since XP accumulates across lessons and packs.
