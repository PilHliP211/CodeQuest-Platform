# S-12.03: Per-Lesson Progress Save/Restore

**Epic:** E-12
**Size:** S
**Dependencies:** S-12.02 (`progressStore`), S-11.06 (LessonContext)

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `unit-tests` — Vitest AAA+O pattern for the pure logic added here
- `localstorage-pattern` — save on state change, restore on load
- `react-context-pattern` — side effects in `useEffect` watching lesson state
- `typescript-standards` — typed progress objects

## What

When lesson phase state changes (a phase completes, syntax is unlocked, a flag is collected), save the updated progress to localStorage. When the lesson first loads, restore any existing progress so a page refresh resumes at the correct phase.

This story wires `progressStore` into `LessonContext` via `useEffect`.

## Tasks

1. In `LessonContext.tsx` (or a sibling `useProgressSync.ts` hook), add a `useEffect` that saves progress whenever `lessonState` changes:
   ```ts
   const { lessonState } = useLessonRunner();
   const { course } = useContent();

   useEffect(() => {
     const progress: LessonProgress = {
       phase1Complete: lessonState.currentPhase > 1 || lessonState.currentPhase === 'complete',
       phase2Complete: lessonState.currentPhase > 2 || lessonState.currentPhase === 'complete',
       phase3Complete: lessonState.currentPhase === 'complete',
       syntaxUnlocked: lessonState.syntaxUnlocked,
       flagCollected: lessonState.currentPhase === 'complete',
       xpEarned: 0, // Updated by xpCalculator separately
       completedAt: lessonState.currentPhase === 'complete' ? new Date().toISOString() : undefined,
     };
     saveLessonProgress(course.id, lessonState.lessonId, progress);
   }, [lessonState.currentPhase, lessonState.syntaxUnlocked]);
   ```

2. On `LessonProvider` mount, load existing progress and initialize `lessonState` from it. Pass the saved progress to `createInitialState` as an optional parameter:
   ```ts
   const [lessonState, dispatch] = useReducer(
     lessonReducer,
     { lessonId: lesson.id, savedProgress: getLessonProgress(course.id, lesson.id) },
     ({ lessonId, savedProgress }) => initFromProgress(lessonId, savedProgress),
   );
   ```
   `initFromProgress` sets `currentPhase` based on `savedProgress` flags:
   - `phase3Complete` → `currentPhase: 'complete'`, skip all narratives
   - `phase2Complete` → `currentPhase: 3`, `syntaxUnlocked: true`
   - `phase1Complete` → `currentPhase: 2`
   - Otherwise → `currentPhase: 1`

3. When restoring to phase 2 or 3, set `phaseStatus: 'challenge'` directly (skip the intro narrative — the player has already seen it).

## Done When

- [ ] `saveLessonProgress` called when phase or `syntaxUnlocked` changes
- [ ] Lesson resumes at the correct phase after page refresh
- [ ] Restored lesson skips intro narratives (sets `phaseStatus: 'challenge'`)
- [ ] `completedAt` set when `phase3Complete` becomes true
- [ ] `npx tsc --noEmit` and `npm run lint` pass
