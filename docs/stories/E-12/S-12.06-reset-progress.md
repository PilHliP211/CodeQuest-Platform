# S-12.06: Reset Progress in Settings (With Confirmation)

**Epic:** E-12
**Size:** S
**Dependencies:** S-12.02 (progressStore `resetAllProgress`), S-02.13 (settings screen)

- `component-tests` ŌĆö Testing Library behavior tests for the component

## What

The settings screen (E-02) gets a "Reset Progress" button. Because this action is irreversible (all phase flags, XP, and flag collection are deleted), it requires an explicit confirmation dialog. The player's name is preserved.

Do not use `window.confirm()` ŌĆö it looks terrible on mobile and is not accessible. Use a simple inline confirmation UI.

## Tasks

1. Add a "Danger Zone" section to `SettingsScreen.tsx` with a reset button:
   ```tsx
   {showConfirm ? (
     <div role="alertdialog" aria-modal="true" aria-labelledby="confirm-title">
       <p id="confirm-title" className="font-pixel text-sm">
         Erase all progress? Your name will stay, but all completed levels and XP will be gone.
       </p>
       <button type="button" onClick={handleConfirmReset} className="text-red-400 font-pixel">
         Yes, reset everything
       </button>
       <button type="button" onClick={() => setShowConfirm(false)} className="font-pixel">
         Cancel
       </button>
     </div>
   ) : (
     <button
       type="button"
       onClick={() => setShowConfirm(true)}
       className="text-red-400 font-pixel text-sm"
     >
       Reset Progress
     </button>
   )}
   ```

2. `handleConfirmReset`:
   ```ts
   function handleConfirmReset(): void {
     resetAllProgress();
     setShowConfirm(false);
     // Navigate back to the map ŌĆö the map will now show all nodes as locked/available per defaults
     onNavigateToMap();
   }
   ```
   `onNavigateToMap` is a prop from the parent, or use the app navigation mechanism established in E-11.

3. The confirmation dialog should be keyboard-accessible: `Escape` key cancels (via `useEffect` + `keydown` listener), and the "Cancel" button receives focus when the dialog opens (`autoFocus`).

4. After reset, `getTotalXP()` returns 0 and all `deriveNodeStates` returns `'available'` for `initiallyUnlocked` nodes and `'locked'` for the rest ŌĆö this happens automatically since the store is cleared.
