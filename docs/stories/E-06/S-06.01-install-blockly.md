## What

Install the `blockly` npm package and create the foundational theme file that all Blockly-related code will import. The theme file establishes the palette (colors, fonts) so that every story in E-06 builds on a consistent visual foundation from day one — not retrofitted at the end.

Blockly ships its own TypeScript definitions since v9. No separate `@types/blockly` package is needed. Confirm the installed version includes `blockly/core` and `blockly/javascript` sub-paths (required for tree-shaking in Vite).

## Tasks

1. Install Blockly:
   ```bash
   npm install blockly
   ```
   Pin to a specific minor version (e.g. `^11.0.0`). Record the version in a comment at the top of `blocklyTheme.ts` so future upgrades are intentional.

2. Confirm Vite can import Blockly without issues. Add a minimal smoke-test import in a scratch file, run `npm run build`, then remove the scratch file. Blockly is a CommonJS package — Vite may need `optimizeDeps.include: ['blockly']` in `vite.config.ts`:
   ```ts
   // vite.config.ts
   export default defineConfig({
     optimizeDeps: {
       include: ['blockly'],
     },
   });
   ```

3. Create `src/editor/blocklyTheme.ts`:
   ```ts
   import * as Blockly from 'blockly/core';

   /**
    * CodeQuest pixel-art Blockly theme.
    * Applied in BlockEditor.tsx via Blockly.inject() options.
    *
    * Blockly version: ^11.0.0
    */
   export const codeQuestTheme = Blockly.Theme.defineTheme('codequest', {
     name: 'codequest',
     base: Blockly.Themes.Classic,
     blockStyles: {
       movement_blocks: { colourPrimary: '160', colourSecondary: '120', colourTertiary: '90' },
       search_blocks:   { colourPrimary: '210', colourSecondary: '180', colourTertiary: '150' },
       drawing_blocks:  { colourPrimary: '20',  colourSecondary: '350', colourTertiary: '330' },
       logic_blocks:    { colourPrimary: '210', colourSecondary: '180', colourTertiary: '150' },
       loop_blocks:     { colourPrimary: '120', colourSecondary: '90',  colourTertiary: '60'  },
       function_blocks: { colourPrimary: '290', colourSecondary: '260', colourTertiary: '230' },
     },
     componentStyles: {
       workspaceBackgroundColour: '#1a1a2e',
       toolboxBackgroundColour:   '#16213e',
       toolboxForegroundColour:   '#ffffff',
       flyoutBackgroundColour:    '#0f3460',
       flyoutForegroundColour:    '#e0e0e0',
       flyoutOpacity:             0.9,
       scrollbarColour:           '#ffffff',
       insertionMarkerColour:     '#ffffff',
       insertionMarkerOpacity:    0.3,
       scrollbarOpacity:          0.4,
       cursorColour:              '#d0d0ff',
     },
     fontStyle: {
       family: '"Press Start 2P", monospace',
       weight: 'normal',
       size: 8,
     },
   });
   ```

4. Document in a comment which Blockly block categories map to which `blockStyles` key. Lesson block definitions in content JSON use a `category` field (`"Movement"`, `"Drawing"`, etc.) — `blockRegistry.ts` will need to map those to the matching style key.
