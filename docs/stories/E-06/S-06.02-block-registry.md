**Epic:** E-06
**Size:** S
**Dependencies:** S-06.01 (Blockly installed), S-03.01 (`BlockDef` type in `src/types/content.ts`)

## What

`blockRegistry.ts` reads the lesson's `blocks` array and registers each block definition with Blockly's block factory and JavaScript code generator. It is called once when a lesson loads (from `BlockEditor.tsx` on mount).

Two registration steps are needed for each block:
1. **Shape registration** — `Blockly.defineBlocksWithJsonArray([def.blocklyDef])` tells Blockly the block's visual shape, inputs, and connections.
2. **Code generator registration** — `Blockly.JavaScript.forBlock[def.id] = () => def.code + '\n'` tells Blockly's JavaScript generator what simplified-JS string this block produces.

## Tasks

1. Ensure `BlockDef` is defined in `src/types/content.ts` (check E-03 deliverables; add if missing):
   ```ts
   export interface BlockDef {
     id: string;
     label: string;
     category: string;
     color: number;
     code: string;
     blocklyDef: Record<string, unknown>;
   }
   ```

2. Create `src/editor/blockRegistry.ts`:
   ```ts
   import * as Blockly from 'blockly/core';
   import { javascriptGenerator } from 'blockly/javascript';
   import type { BlockDef } from '@/types/content';

   /** Maps content-pack category names to Blockly block style keys. */
   const CATEGORY_STYLE_MAP: Record<string, string> = {
     Movement:         'movement_blocks',
     'Search / Collect': 'search_blocks',
     Drawing:          'drawing_blocks',
     Logic:            'logic_blocks',
     Loops:            'loop_blocks',
     Functions:        'function_blocks',
   };

   /**
    * Registers an array of BlockDef entries from the lesson content pack
    * with Blockly's block factory and JavaScript code generator.
    *
    * Call this once per lesson load, before injecting the workspace.
    * Safe to call again with the same blocks — Blockly overwrites duplicate ids.
    */
   export function registerBlocks(blocks: BlockDef[]): void {
     const blocklyDefs = blocks.map((def) => ({
       ...def.blocklyDef,
       type: def.id,
       style: CATEGORY_STYLE_MAP[def.category] ?? 'movement_blocks',
     }));

     Blockly.defineBlocksWithJsonArray(blocklyDefs);

     for (const def of blocks) {
       javascriptGenerator.forBlock[def.id] = () => [def.code + '\n', 0] as [string, number];
     }
   }
   ```

3. Note on the code generator return value: Blockly's JavaScript generator expects either a `string` (for statement blocks) or a `[string, number]` tuple (for expression blocks). Movement/drawing blocks are statement blocks — return `def.code + '\n'` as a plain string:
   ```ts
   javascriptGenerator.forBlock[def.id] = () => def.code + '\n';
   ```
   Confirm the correct return type with the installed Blockly version's type definitions. The `+'\n'` ensures each block generates on its own line.

4. The `code` field from the content pack is a pre-authored string (e.g. `"moveEast()"`). `blockRegistry.ts` never constructs or evaluates code — it only copies the static string into the generator. This is safe by design.
