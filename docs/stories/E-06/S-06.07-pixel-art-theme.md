# S-06.07: Pixel-Art Blockly Theme

**Epic:** E-06
**Size:** S
**Dependencies:** S-06.01 (Blockly installed, `blocklyTheme.ts` skeleton)

## Skills to Load

- `naming-conventions` — `blocklyTheme.ts` lives in `src/editor/`
- `formatting` — pixel-art aesthetic: squared corners, no drop shadows, pixel font

## What

Blockly ships with a default theme that looks like a web app from 2014. This story replaces it with a pixel-art theme that matches the CodeQuest visual identity: squared corners, Press Start 2P font for block labels, category colors matching the block color table in CONTENT_SCHEMA.md, and a dark toolbox background.

The theme object is created once in `blocklyTheme.ts` and passed to `Blockly.inject()` options in `BlockEditor.tsx`.

## Tasks

1. Fill out `src/editor/blocklyTheme.ts`:
   ```ts
   import * as Blockly from 'blockly/core';

   export const codeQuestTheme = Blockly.Theme.defineTheme('codequest', {
     base: Blockly.Themes.Classic,
     componentStyles: {
       workspaceBackgroundColour: '#1a1a2e',
       toolboxBackgroundColour: '#16213e',
       toolboxForegroundColour: '#e0e0e0',
       flyoutBackgroundColour: '#0f3460',
       flyoutForegroundColour: '#e0e0e0',
       flyoutOpacity: 0.95,
       scrollbarColour: '#4a4a6a',
       insertionMarkerColour: '#ffffff',
       insertionMarkerOpacity: 0.3,
       cursorColour: '#ffffff',
       blackBackground: '#1a1a2e',
     },
     fontStyle: {
       family: '"Press Start 2P", monospace',
       weight: 'normal',
       size: 8,
     },
     startHats: false,
   });
   ```

2. Blockly's built-in font rendering squishes "Press Start 2P" at small sizes. If labels are unreadable, fall back to `'"Courier New", monospace'` at size 10 and document the decision. The priority is readable block labels, not pixel-aesthetic purity.

3. Block category colors are already encoded in the `BlockDef.color` field (e.g. Movement=160, Drawing=20). These are Blockly hue values (0-360) and are set per-block via the `blocklyDef.colour` field — the theme does not need to override them.

4. Ensure `image-rendering: pixelated` is applied to the Blockly SVG container via CSS:
   ```css
   /* In src/index.css or equivalent */
   .blocklyMainBackground,
   .blocklyFlyoutBackground {
     image-rendering: pixelated;
   }
   ```
   Note: Blockly renders as SVG, not canvas. `image-rendering: pixelated` applies mainly to embedded images in blocks (if any). The primary visual effect here comes from the color scheme and font.
