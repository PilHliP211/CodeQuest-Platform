## What

A "Run ‚ñ∂" button sits below the block editor. When clicked, it calls `Blockly.JavaScript.workspaceToCode(workspace)` to generate the simplified-JS code string from the current workspace, then fires the `onCodeGenerated(code)` callback so the parent (lesson runner, E-11) can pass it to the interpreter (E-09).

The button is disabled while code is running (`isRunning` prop) to prevent double-execution. A loading indicator shows during execution.

## Tasks

1. Add a `RunButton.tsx` component in `src/editor/`:
   ```tsx
   interface RunButtonProps {
     isRunning: boolean;
     onRun: () => void;
   }

   export function RunButton({ isRunning, onRun }: RunButtonProps): React.ReactElement {
     return (
       <button
         type="button"
         onClick={onRun}
         disabled={isRunning}
         aria-busy={isRunning}
         className="font-pixel px-4 py-2 bg-green-600 text-white disabled:opacity-50 disabled:cursor-not-allowed"
       >
         {isRunning ? 'Running‚Ä¶' : 'Run ‚ñ∂'}
       </button>
     );
   }
   ```

2. Expose `generateCode(): string` from `BlockEditor` via a callback ref or by passing `workspaceRef` upward. The simplest approach: add a `getCode` prop that `BlockEditor` calls when the run button fires. Actually, the cleanest approach: expose `BlockEditor` via `forwardRef` with `useImperativeHandle` and a `getCode(): string` method. (S-06.06 will extend the same imperative handle for serialization ‚Äî do both in that story if preferred, or scaffold it here.)

   Simpler alternative for now: move the run button inside `BlockEditor.tsx` as an internal button, and wire it directly to `workspaceRef`:
   ```tsx
   function handleRun(): void {
     if (workspaceRef.current === null) return;
     const code = Blockly.JavaScript.workspaceToCode(workspaceRef.current);
     onCodeGenerated(code);
   }
   ```
   Add `isRunning: boolean` to `BlockEditorProps`. Render `<RunButton isRunning={isRunning} onRun={handleRun} />` below the workspace div.

3. Import `'blockly/javascript'` (or `blockly/generators/javascript`) ‚Äî verify the correct import path for the installed Blockly version.

4. If the workspace is empty, `workspaceToCode` returns an empty string `""`. Treat this as valid ‚Äî the interpreter will handle it (no-op execution).
