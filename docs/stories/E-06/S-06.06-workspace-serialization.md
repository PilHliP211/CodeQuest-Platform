# S-06.06: Workspace State Serialization (Save/Restore)

**Epic:** E-06
**Size:** S
**Dependencies:** S-06.03 (`BlockEditor.tsx`)

## What

Blockly workspaces need to survive mid-lesson navigation (player goes to settings and back). This story exposes `getState()` and `setState()` on `BlockEditor` via `useImperativeHandle`, letting the lesson runner (E-11) save/restore workspace JSON to/from the progress store (E-12) without coupling `BlockEditor` to persistence logic.

## Tasks

1. Define the handle interface:
   ```ts
   export interface BlockEditorHandle {
     getState: () => Record<string, unknown>;
     setState: (state: Record<string, unknown>) => void;
   }
   ```

2. Convert `BlockEditor` to use `forwardRef`:
   ```tsx
   export const BlockEditor = forwardRef<BlockEditorHandle, BlockEditorProps>(
     function BlockEditor(props, ref) {
       // ... existing implementation ...

       useImperativeHandle(ref, () => ({
         getState(): Record<string, unknown> {
           if (workspaceRef.current === null) return {};
           return Blockly.serialization.workspaces.save(
             workspaceRef.current,
           ) as Record<string, unknown>;
         },
         setState(state: Record<string, unknown>): void {
           if (workspaceRef.current === null) return;
           Blockly.serialization.workspaces.load(state, workspaceRef.current);
         },
       }));

       // ... rest of render ...
     },
   );
   ```

3. The `initialState` prop (already on `BlockEditorProps`) still handles the mount-time restore. `setState` via the ref handle is for imperative mid-session restores. Both use `Blockly.serialization.workspaces.load` internally.

4. In `BlockEditor`, move the `initialState` loading from the `useEffect` to a call to the handle's `setState` after workspace injection — this keeps the logic in one place.

5. The lesson runner (E-11) will call `editorRef.current?.getState()` before navigating away and `editorRef.current?.setState(saved)` on return. This story just provides the API; the calling side is implemented in E-11/E-12.
