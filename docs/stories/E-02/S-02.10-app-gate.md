# S-02.10: App Gate

**Epic:** E-02
**Size:** XS
**Dependencies:** S-02.09

## Skills to Load

- `react-component-structure` — conditional rendering
- `react-context-pattern` — using the hook to drive routing decisions

## What

In `App.tsx`, render the `NameEntryScreen` when `profile === null`. Otherwise render the rest of the app (currently a placeholder; later epics fill it in). This pattern — a "gate" component that blocks the app until a precondition is met — repeats in E-03 for content load errors.

## Tasks

1. Update `App.tsx`:
   ```tsx
   import { useProfile } from '@/engine/ProfileContext';
   import { NameEntryScreen } from '@/components/Profile/NameEntryScreen';

   export function App(): React.ReactElement {
     const { profile } = useProfile();

     if (profile === null) {
       return <NameEntryScreen />;
     }

     return <main>{/* main app placeholder */}</main>;
   }
   ```
2. Verify by clearing localStorage and reloading — name entry should appear. After submitting, the main app should render.

## Done When

- [ ] First launch (no profile) shows `NameEntryScreen`
- [ ] After name entry, main app renders
- [ ] Reloading the page with a profile present skips name entry
- [ ] Clearing localStorage and reloading shows name entry again
