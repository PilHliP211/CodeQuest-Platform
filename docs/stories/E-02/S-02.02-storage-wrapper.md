# S-02.02: Generic localStorage Wrapper

**Epic:** E-02
**Size:** S
**Dependencies:** S-02.01

## Skills to Load

- `localstorage-pattern` — the canonical wrapper shape
- `typescript-standards` — generics and type guards
- `error-handling` — return `null` on failure, never throw
- `naming-conventions` — `storage.ts` lives in `src/engine/`

## What

Create the generic, typed localStorage wrapper that every store in the project will use. This is the most-reused utility in the codebase.

## Tasks

1. Create `src/engine/storage.ts`:
   ```ts
   export function readStorage<T>(
     key: string,
     validate: (data: unknown) => data is T,
   ): T | null {
     try {
       const raw = localStorage.getItem(key);
       if (raw === null) return null;
       const parsed: unknown = JSON.parse(raw);
       return validate(parsed) ? parsed : null;
     } catch {
       return null;
     }
   }

   export function writeStorage<T>(key: string, value: T): void {
     localStorage.setItem(key, JSON.stringify(value));
   }

   export function removeStorage(key: string): void {
     localStorage.removeItem(key);
   }
   ```
2. The wrapper must:
   - Return `null` when the key is missing
   - Return `null` when JSON parse fails (corrupt data)
   - Return `null` when validation fails (wrong shape)
   - Never throw on read

## Done When

- [ ] `src/engine/storage.ts` exists with all three functions exported
- [ ] All functions are typed (no `any`)
- [ ] `readStorage` requires a type guard (no escape hatch)
- [ ] `npm run typecheck` and `npm run lint` pass
