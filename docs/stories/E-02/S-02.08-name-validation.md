# S-02.08: Name Input Validation

**Epic:** E-02
**Size:** XS
**Dependencies:** S-02.07

## Skills to Load

- `react-component-structure` — derived state in components
- `error-handling` — show user-friendly errors inline
- `eslint-rules` — `strict-boolean-expressions` (use `name.trim().length > 0`)

## What

Add validation logic to `NameEntryScreen`:
- Name must be 1-20 characters after trimming
- Whitespace-only names rejected
- Submit disabled when invalid
- Inline validation message shown when input has been touched and is invalid

## Tasks

1. Add derived state to `NameEntryScreen`:
   ```tsx
   const trimmed = name.trim();
   const isValid = trimmed.length > 0 && trimmed.length <= 20;
   const errorMessage =
     trimmed.length === 0 ? null : // don't show error before any input
     trimmed.length > 20 ? 'Name must be 20 characters or fewer' :
     null;
   ```
2. Disable the submit button when `!isValid`
3. Render the error message below the input when present
4. On submit, save with the trimmed name (`createProfile(trimmed)`)

## Done When

- [x] Empty / whitespace-only name cannot be submitted
- [x] Names longer than 20 chars show an error and cannot be submitted
- [x] Submit button is disabled when invalid
- [x] Trimmed name is what gets saved
- [x] Validation is in derived state (no `useEffect` to track validity)
