# S-02.09: Name Entry Form Accessibility

**Epic:** E-02
**Size:** XS
**Dependencies:** S-02.08

## Skills to Load

- `accessibility` — the accessibility minimums
- `react-component-structure` — semantic form elements

## What

Make `NameEntryScreen` accessible: real `<form>` element, labeled input, ARIA attributes for the validation error, autofocus, keyboard submit.

## Tasks

1. Wrap the input + button in a `<form>` with an `onSubmit` handler (call `e.preventDefault()`)
2. Add a `<label htmlFor="name-input">` linked to the input by `id`
3. Mark the input with:
   - `id="name-input"`
   - `aria-invalid={errorMessage !== null}`
   - `aria-describedby={errorMessage !== null ? 'name-error' : undefined}`
   - `autoFocus`
   - `maxLength={40}` (a soft browser-side limit; the real validation lives in code)
4. Render the error message in a `<p id="name-error">` so the ARIA reference resolves
5. Submit button has `type="submit"` and clear text ("Start Quest!")
6. Verify keyboard flow: Tab into input → type → Enter submits

## Done When

- [x] `<form>` element with `onSubmit` (Enter submits)
- [x] Input has a linked `<label>`
- [x] `aria-invalid` and `aria-describedby` set correctly
- [x] Input is auto-focused on mount
- [x] Submit button is type=submit with clear text
- [x] Tab order is correct (input → submit)
