# S-02.03: Profile Type Guard

**Epic:** E-02
**Size:** XS
**Dependencies:** S-02.01

## Skills to Load

- `typescript-standards` — type guard pattern with `unknown`
- `localstorage-pattern` — type guards are required by `readStorage`
- `error-handling` — boundary validation

## What

A `isLearnerProfile` type guard that validates an `unknown` value as a `LearnerProfile`. This is what `readStorage` will call.

## Tasks

1. Create `src/engine/profileGuard.ts` (kept separate from the store so guards are reusable in tests):
   ```ts
   import type { LearnerProfile } from '@/types/profile';

   export function isLearnerProfile(data: unknown): data is LearnerProfile {
     if (typeof data !== 'object' || data === null) return false;
     const record = data as Record<string, unknown>;
     return (
       typeof record.name === 'string' &&
       record.name.length > 0 &&
       typeof record.createdAt === 'string' &&
       !Number.isNaN(Date.parse(record.createdAt))
     );
   }
   ```
2. Validates:
   - `data` is a non-null object
   - `name` is a non-empty string
   - `createdAt` is a parseable ISO 8601 string

## Done When

- [x] `isLearnerProfile` exists and is correctly typed as a type predicate (`data is LearnerProfile`)
- [x] Returns `false` for `null`, `undefined`, primitives, missing fields, wrong types
- [x] Returns `true` for a valid profile object
- [x] No `any` casts (use `Record<string, unknown>` for narrowing)
