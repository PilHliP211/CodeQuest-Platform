# S-02.15: Settings Navigation

**Epic:** E-02
**Size:** XS
**Dependencies:** S-02.11 (HUD), S-02.12 (HUDLayout), S-02.14 (settings form)

## Skills to Load

- `react-component-structure` — local state for view switching
- `accessibility` — button vs link semantics

## What

Wire up the settings screen so the user can open it from the HUD and close it back to the previous view. For v1, use local state in `App.tsx` to switch views — no router needed yet.

## Tasks

1. Add a settings button to the HUD (`HUD.tsx`):
   ```tsx
   interface HUDProps {
     onOpenSettings: () => void;
   }
   ```
   The button uses `<button type="button">` (it's an action, not navigation).
2. Update `HUDLayout` to forward an `onOpenSettings` prop down to `HUD`.
3. In `App.tsx`, manage the settings-open state:
   ```tsx
   const [settingsOpen, setSettingsOpen] = useState(false);

   if (profile === null) return <NameEntryScreen />;
   if (settingsOpen) {
     return <SettingsScreen onClose={() => setSettingsOpen(false)} />;
   }
   return (
     <HUDLayout onOpenSettings={() => setSettingsOpen(true)}>
       {/* main app placeholder */}
     </HUDLayout>
   );
   ```
4. Verify the round trip: HUD button → settings screen → back button → main app.

## Done When

- [ ] HUD has a visible "Settings" button
- [ ] Clicking it shows the settings screen (HUD is hidden — settings is full-screen)
- [ ] Back button closes settings and returns to the main app
- [ ] Changing the name in settings and returning to the main app shows the new name in the HUD
- [ ] Settings button is keyboard-reachable and labeled clearly
