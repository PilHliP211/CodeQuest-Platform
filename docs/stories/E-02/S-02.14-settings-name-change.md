# S-02.14: Settings Name-Change Form

**Epic:** E-02
**Size:** S
**Dependencies:** S-02.13, S-02.08 (validation), S-02.09 (a11y)

## Skills to Load

- `react-component-structure` — derived state, event handlers
- `accessibility` — same form patterns as name entry
- `error-handling` — inline validation

## What

Add the name-change form to the settings screen. Same validation rules and accessibility patterns as `NameEntryScreen` (S-02.07–09). Calls `updateName` from the profile context. **Does not reset progress.**

## Tasks

1. Inside `SettingsScreen`, add a form pre-filled with the current name:
   ```tsx
   const { profile, updateName } = useProfile();
   const [name, setName] = useState(profile?.name ?? '');
   const [savedFlash, setSavedFlash] = useState(false);

   const trimmed = name.trim();
   const isValid = trimmed.length > 0 && trimmed.length <= 20;
   const isChanged = profile !== null && trimmed !== profile.name;

   const handleSubmit = (e: React.FormEvent): void => {
     e.preventDefault();
     if (!isValid || !isChanged) return;
     updateName(trimmed);
     setSavedFlash(true);
     window.setTimeout(() => setSavedFlash(false), 1500);
   };
   ```
2. Same `<form>` + `<label>` + `aria-invalid` + `aria-describedby` pattern as the name entry screen.
3. Save button disabled when invalid OR unchanged.
4. Show a brief "Saved!" confirmation when the save succeeds.
5. **Crucially:** updating the name must NOT touch any other localStorage keys. Only `codequest:profile`.

## Done When
