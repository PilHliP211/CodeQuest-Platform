# S-02.05: ProfileContext + Provider

**Epic:** E-02
**Size:** S
**Dependencies:** S-02.04

## Skills to Load

- `react-context-pattern` — the canonical provider/hook shape
- `react-component-structure` — file layout, named exports
- `localstorage-pattern` — provider does its own persistence
- `typescript-standards` — `useMemo` typing

## What

The `ProfileProvider` component holds profile state in React, reads from `profileStore` on mount, and exposes `createProfile` / `updateName` actions. The hook (`useProfile`) is in S-02.06.

## Tasks

1. Create `src/engine/ProfileContext.tsx`:
   ```tsx
   import { createContext, useMemo, useState, type ReactNode } from 'react';
   import type { LearnerProfile } from '@/types/profile';
   import { loadProfile, saveProfile } from './profileStore';

   export interface ProfileContextValue {
     profile: LearnerProfile | null;
     createProfile: (name: string) => void;
     updateName: (name: string) => void;
   }

   export const ProfileContext = createContext<ProfileContextValue | undefined>(undefined);

   export function ProfileProvider({ children }: { children: ReactNode }): React.ReactElement {
     const [profile, setProfile] = useState<LearnerProfile | null>(() => loadProfile());

     const createProfile = (name: string): void => {
       const next: LearnerProfile = { name, createdAt: new Date().toISOString() };
       saveProfile(next);
       setProfile(next);
     };

     const updateName = (name: string): void => {
       if (profile === null) return;
       const next: LearnerProfile = { ...profile, name };
       saveProfile(next);
       setProfile(next);
     };

     const value = useMemo<ProfileContextValue>(
       () => ({ profile, createProfile, updateName }),
       [profile],
     );

     return <ProfileContext.Provider value={value}>{children}</ProfileContext.Provider>;
   }
   ```
2. Wrap `<App />` in `<ProfileProvider>` (likely in `main.tsx`)

## Done When

- [ ] Provider reads from localStorage on mount via `useState` initializer
- [ ] `createProfile` writes to localStorage and updates state
- [ ] `updateName` preserves `createdAt` and only mutates `name`
- [ ] Context value is memoized (no re-render storms)
- [ ] App is wrapped in the provider
