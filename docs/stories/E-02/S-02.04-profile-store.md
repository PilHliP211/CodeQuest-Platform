# S-02.04: Profile Store

**Epic:** E-02
**Size:** XS
**Dependencies:** S-02.02 (storage wrapper), S-02.03 (type guard)

## Skills to Load

- `localstorage-pattern` — per-domain store shape
- `naming-conventions` — `codequest:` key prefix

## What

The thin profile-specific wrapper around `storage.ts`. Provides `loadProfile` / `saveProfile` operations the rest of the app uses.

## Tasks

1. Create `src/engine/profileStore.ts`:
   ```ts
   import type { LearnerProfile } from '@/types/profile';
   import { readStorage, writeStorage, removeStorage } from './storage';
   import { isLearnerProfile } from './profileGuard';

   const PROFILE_KEY = 'codequest:profile';

   export function loadProfile(): LearnerProfile | null {
     return readStorage(PROFILE_KEY, isLearnerProfile);
   }

   export function saveProfile(profile: LearnerProfile): void {
     writeStorage(PROFILE_KEY, profile);
   }

   export function clearProfile(): void {
     removeStorage(PROFILE_KEY);
   }
   ```

## Done When

- [ ] `profileStore.ts` exists with `loadProfile`, `saveProfile`, `clearProfile`
- [ ] Uses the `codequest:profile` key
- [ ] Goes through `storage.ts` (does not call `localStorage` directly)
- [ ] All functions have explicit return types
