# S-14.08: Asset Sourcing and ATTRIBUTION.md

**Epic:** E-14
**Size:** S
**Dependencies:** S-14.01 (directory structure created)

## Skills to Load

- `testing-strategy` — background on the outcome-focused validation philosophy; this story adds no executable code, so no new tests are required
- `content-pack-system` — asset path conventions

## What

Source or create the visual assets needed for the Japan lesson. All assets must be CC-licensed (or created from scratch). Attribution is tracked in `content/flag-hunter/ATTRIBUTION.md`.

The Japan flag itself (`assets/flags/japan.png`) can be created programmatically — it's just a white rectangle with a red circle — so no external asset is required for it. Other assets (world map, player sprite, villain sprite) require finding or creating CC-licensed pixel art.

## Tasks

1. **Japan flag icon** — create `content/flag-hunter/assets/flags/japan.png` as a simple 64×32 PNG:
   - White background
   - Red circle centered at (32, 16), radius 12
   - Can be generated with a small script or any image editor
   - No external asset needed; record as "Created for CodeQuest Platform" in ATTRIBUTION.md

2. **World map background** (`assets/world-map.png`) — search OpenGameArt.org for a CC-licensed pixel-art world map:
   - Search terms: `pixel art world map`, `top-down world map`, `game world map`
   - Preferred: CC0 or CC-BY license
   - Target size: approximately 1200×600 px
   - If no suitable asset is found, use a solid color placeholder (`#1a3a5c` dark blue) and note it's a placeholder

3. **Player sprite** (`assets/player.png`) — search OpenGameArt.org:
   - Search terms: `pixel art character sprite`, `top-down character`, `pixel character 16x16`
   - Must be CC0 or CC-BY
   - A simple 16×16 or 32×32 top-down character

4. **Villain sprite** (`assets/villain.png`) and **trail** (`assets/villain-trail.png`) — same search criteria as player sprite. The trail sprite can be the villain sprite at 50% opacity, or a separate "footprints" icon.

5. **Portrait images** (`assets/player-portrait.png`, lesson-specific `assets/guide-portrait.png`) — 64×64 or 128×128 character portraits. Can be the same sprite scaled up, or separate portrait art.

6. **Phase background images** (`lessons/001-japan/assets/phase1-bg.png`, etc.) — pixel-art Japan scene backgrounds. Search OpenGameArt.org for CC-licensed Japan-themed backgrounds. Alternatively, use solid color placeholders.

7. Create `content/flag-hunter/ATTRIBUTION.md`:
   ```markdown
   # Asset Attribution

   ## Created for CodeQuest Platform
   - `assets/flags/japan.png` — Drawn programmatically; no external source.

   ## OpenGameArt.org (replace placeholders with actual sources)
   - `assets/world-map.png` — [ASSET NAME] by [AUTHOR], [LICENSE], [URL]
   - `assets/player.png` — [ASSET NAME] by [AUTHOR], [LICENSE], [URL]
   - `assets/villain.png` — [ASSET NAME] by [AUTHOR], [LICENSE], [URL]
   - `assets/villain-trail.png` — [derived from villain.png]
   ```
