# S-14.01: `course.json` Scaffold

**Epic:** E-14
**Size:** S
**Dependencies:** S-03.01 (content types — `Course` interface for schema reference)

## Skills to Load

- `testing-strategy` — background on the outcome-focused validation philosophy; this story adds no executable code, so no new tests are required
- `content-pack-system` — `course.json` schema per CONTENT_SCHEMA.md
- `naming-conventions` — content pack root at `content/flag-hunter/`

## What

Create the top-level `course.json` for the Flag Hunter content pack. This file defines everything the platform needs to know about the course: metadata, world map configuration (1 node for Japan), XP rewards, villain and player config.

## Tasks

1. Create `content/flag-hunter/course.json`:
   ```json
   {
     "id": "flag-hunter",
     "title": "CodeQuest: Flag Hunter",
     "description": "A mysterious villain is erasing countries from history. Chase her across the globe and restore each nation's flag — one line of code at a time.",
     "version": "1.0.0",
     "map": {
       "backgroundImage": "assets/world-map.png",
       "width": 1200,
       "height": 600,
       "startNodeId": "001-japan",
       "nodes": [
         {
           "id": "001-japan",
           "label": "Japan",
           "x": 0.82,
           "y": 0.28,
           "unlocksOnComplete": [],
           "initiallyUnlocked": true,
           "collectedIcon": "assets/flags/japan.png"
         }
       ],
       "edges": []
     },
     "lessons": [
       {
         "id": "001-japan",
         "path": "lessons/001-japan/lesson.json"
       }
     ],
     "xp": {
       "phaseComplete": 10,
       "lessonComplete": 50,
       "hintUsed": -5,
       "syntaxEditorUsed": 5
     },
     "villain": {
       "name": "??",
       "sprite": "assets/villain.png",
       "trailSprite": "assets/villain-trail.png"
     },
     "player": {
       "sprite": "assets/player.png"
     }
   }
   ```

2. Create the content pack directory structure:
   ```
   content/
   └── flag-hunter/
       ├── course.json
       ├── ATTRIBUTION.md        (created in S-14.08)
       ├── assets/
       │   ├── world-map.png     (placeholder — source in S-14.08)
       │   ├── player.png        (placeholder)
       │   ├── villain.png       (placeholder)
       │   ├── villain-trail.png (placeholder)
       │   └── flags/
       │       └── japan.png     (placeholder)
       └── lessons/
           └── 001-japan/
               ├── lesson.json   (S-14.02)
               └── assets/       (S-14.08)
   ```

3. For placeholders, use 1×1 pixel transparent PNGs or simple colored rectangles until real assets are sourced in S-14.08. The platform should not crash if images are missing — they'll just show as broken image icons.

4. Verify `isCourse(courseJson)` passes on this JSON using the type guard from E-03 (S-03.02).

## Done When

- [ ] `content/flag-hunter/course.json` exists and is valid JSON
- [ ] `isCourse` type guard passes on the JSON
- [ ] All directory structure created
- [ ] 1 node (`001-japan`) at coordinates (0.82, 0.28)
- [ ] `initiallyUnlocked: true` on the Japan node
- [ ] XP config matches the values above
