# S-14.09: End-to-End Playthrough Verification

**Epic:** E-14
**Size:** M
**Dependencies:** All E-01 through E-14 stories complete

## Skills to Load

- `testing-strategy` — **required** — outcome-focused validation before writing any test
- `e2e-tests` — Playwright end-to-end for this critical learner journey
- `ci-and-quality-gates` — manual QA as a quality gate before v1 ship

## What

This is a manual test checklist. A developer plays through the complete Japan lesson from scratch — clearing localStorage first — and checks off every AC from the PRD (F-01 through F-09). No automated test can substitute for this: it validates the end-to-end integration of every subsystem.

If any AC fails, it must be fixed before this story is marked Done.

## Playthrough Steps

Follow these steps in order, checking each box as you go:

### Setup
- [ ] Clear localStorage completely (DevTools → Application → Storage → Clear All)
- [ ] Run `npm run dev` and open the app in a browser
- [ ] Confirm the app loads without console errors

### F-01: Learner Profile
- [ ] Name entry screen appears on first launch (no game content visible)
- [ ] Entering a name and submitting shows the main app (map screen)
- [ ] Learner's name is visible in the HUD
- [ ] Reloading the page skips name entry and goes straight to the map
- [ ] Settings screen has a name-change form that works without resetting progress

### F-02: Content Pack Loading
- [ ] App loads without errors — content pack is valid
- [ ] Map is rendered with the Japan node visible
- [ ] Content pack error gate would show if `course.json` were malformed (not a ship blocker — test in dev by temporarily breaking the JSON)

### F-03: World Map
- [ ] Japan node is visible on the map at the correct position
- [ ] Node state is "available" (highlighted, clickable)
- [ ] Clicking the Japan node starts the lesson
- [ ] Map background image loads (or placeholder visible if not sourced yet)

### F-04: Phase 1 (Navigate)
- [ ] Phase 1 intro narrative plays with correct dialogue
- [ ] Concept intro is shown before the challenge
- [ ] Only the 4 movement blocks are in the palette
- [ ] Running correct code (`moveEast()`, `moveEast()`, `moveNorth()`) advances to Phase 2
- [ ] Running incorrect code shows an error message and allows retry
- [ ] Hint appears after 2 failed attempts

### F-05: Phase 2 (Investigate)
- [ ] Phase 2 intro narrative plays with Hana's dialogue
- [ ] Blocks display their code syntax inside the block body
- [ ] Running the correct sequence finds both items
- [ ] Reveal narrative plays after solving
- [ ] Phase 2 completion permanently sets `syntaxUnlocked: true` in localStorage

### F-06: Phase 3 (Restore)
- [ ] Phase 3 intro narrative plays
- [ ] Syntax editor is the primary interface (not blocks)
- [ ] "View in Blocks" toggle is available and shows read-only blocks
- [ ] Running `fillBackground("white")` and `drawCircle(150, 100, 60, "red")` succeeds
- [ ] Success triggers celebration animation and flag reveal
- [ ] After 3 failed attempts, "Need help? Try blocks →" button appears

### F-07: Flag Collection
- [ ] Japan flag appears in the collection after lesson completion
- [ ] Flag is shown in full color (not silhouette)
- [ ] Collection counter shows "1 / 1 Flags Collected"
- [ ] Completion date is shown on hover/focus

### F-08: XP & Progression
- [ ] XP appears in the HUD after each phase completes
- [ ] XP persists after page reload
- [ ] Japan node shows as "completed" on the map after the lesson
- [ ] Progress persists after browser restart (close and reopen)

### F-09: Deployment (if applicable)
- [ ] `npm run build` produces a valid static build
- [ ] Build works when served from a subpath (`/codequest-platform/`)

### Final Sign-Off
- [ ] No console errors during the full playthrough
- [ ] No TypeScript errors (`npx tsc --noEmit` passes)
- [ ] `npm run lint` passes
- [ ] All linters and build checks pass in CI

## Done When

All checkboxes above are checked. Any failing check is a bug that must be fixed before this story is closed.
